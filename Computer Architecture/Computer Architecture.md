## 第四章 指令系统结构

1. （1）堆栈型指令系统：堆栈型指令系统又称为**零地址指令系统**。它的操作数都隐含在**栈顶**，在运算指令中不需要显式指定操作数。例如执行加法指令时，系统默认将栈顶的两个操作数相加，并把运算结果重新压回栈顶。该指令系统的特点是指令格式简单，但对栈的访问频繁，数据通路受限。

   （2）累加器型指令系统：累加器型指令系统又称为**单地址指令系统**。它的一个操作数是**隐含的累加器**，另一个操作数由指令显式给出，运算结果通常也写回到累加器中。由于累加器是固定的隐含操作数，指令长度较短，但运算过程中对累加器的依赖较强，灵活性有限。

   （3）寄存器—存储器型指令系统：寄存器—存储器型指令系统中，**每个操作数都由指令显式指定**，操作数可以是寄存器，也可以是存储器单元地址。运算时，一个操作数来自寄存器，另一个操作数来自存储器，运算结果通常存入寄存器。该指令系统减少了部分访存次数，但指令格式相对复杂。

   （4）寄存器—寄存器型指令系统：寄存器—寄存器型指令系统中，**所有操作数都来自寄存器**，每个操作数均由指令显式指定，运算结果也存放在寄存器中（访存仅通过专门的访存指令完成）。该系统的指令执行速度快、数据通路规整，是现代 RISC 处理器常用的指令系统结构，但对寄存器数量要求较高。

## 第六章 动态流水线

1. 保留站：指令有序进入保留站，在保留站内等待相关被解决后乱序发射至功能部件进行执行

   重命名寄存器：通过对同一个逻辑寄存器重命名成多个不同的物理寄存器，可以解决WAW相关，并且避免伪RAW相关

   Reorder buffer：指令不论何时写回寄存器，它在reorder buffer中退出的顺序必须符合程序。这保证了乱序执行下，中断能有精确现场

   ![image-20260122183030866](C:\Users\86150\AppData\Roaming\Typora\typora-user-images\image-20260122183030866.png)

3. **在Tomasulo算法中**：通过寄存器重命名技术，消除了`WAR`和`WAW`相关。在Tomasulo算法中，寄存器重命名的功能是通过保留站来实现的。当指令发射到保留站时，它的目标寄存器对应保留站号，消除了WAR相关和WAW相关。如果指令的某操作数没有准备好，在它的保留站中的该操作数的位置上标记的是产生该操作数的指令的保留站号。只有当操作数都准备好了，指令才能执行，保证了RAW相关

   **在记分板技术中**：在指令发射阶段，如果存在WAW相关，则停止发射该指令和任何后继指令。如果指令的源操作数都准备好了，则计分板会通知相应功能部件读取该操作数并开始执行该指令。这样保证了RAW相关。在指令写回阶段，如果存在WAR相关，则停止写回

4. **精确例外**：就是指在处理例外的时候，发生例外指令之前所有的指令都已经执行完了，例外指令后面的所有指令都还没有执行（严格的说是没有产生执行效果，即修改处理机状态)

   实现精确例外处理的办法：重排序缓存(Reorder Buffer，简称ROB) 通过将指令执行结果暂存于按程序顺序排列的缓冲区，并仅在提交阶段顺序地更新体系结构状态，在发生异常时丢弃异常指令及其后的所有指令结果，从而保证处理器状态等价于顺序执行到异常指令为止，实现精确例外。**本质上是乱序执行，顺序提交**

5. （1）循环展开：**根据题目**确定冲突指令间的最小指令数，通过多循环进行填充，并**消除分支指令后的 `NOP`**

   （2）软件流水：a. 画出数据流图，判断数据冲突；对于循环中自增/测试/分支指令，一般无数据冲突，**数据冲突只存在加载/运算/存储操作中**；b. 依据单循环的执行顺序**初步**确定填充/主体/排空；c. 尽量从第一组最后一个开始，划归循环主体。在划分循环主体时，要**①注意寄存器重命名**（可以通过填充流水确定）；**②同组内的数据冲突**；③同组内如果被划归到循环主体，要顺序执行；**④偏移量值的计算**（要进行溯源确认，同时记得自增后偏移量的改变）；**⑤如果出现“未用但要覆盖的情况”，要把覆盖的指令放置到循环主体中**；d. 循环主体添加自增/测试/分支指令，并**消除分支指令后的 `NOP`**

6. 在填写寄存器结果时的注意事项：（1）保留站依次是：指令，操作数1是否就绪（就绪为 `0`，**否则为需要等待的指令编号**），操作数1的值（未就绪则空），操作数2是否就绪，操作数2的值；（2）**要着重注意指令队列中数据相关的指令**；（3）指令操作流程为：指令队列发射 — 写入 ROB — 写入保留站 — 从寄存器中取数并占用寄存器 — **运算结束后写回 ROB** — 按顺序写回浮点寄存器

```verilog
// =======================================================
// 模块1：4位 CLA 子模块
// =======================================================
module cla_4 (
    input  [3:0] a, b,   // 操作数
    input        cin,     // 进位输入
    output [3:0] sum,     // 和
    output       P, G     // 组传递、组生成
);

    wire [3:0] p, g;
    wire [3:0] c;

    // 1. 位级 p、g 计算
    assign g = a & b;
    assign p = a ^ b;

    // 2. 组 PG 计算
    assign P = &p;
    assign G = g[3] | (p[3] & g[2]) 
                    | (p[3] & p[2] & g[1]) 
                    | (p[3] & p[2] & p[1] & g[0]);

    // 3. 组内进位计算
    assign c[0] = cin;
    assign c[1] = g[0] | (p[0] & cin);
    assign c[2] = g[1] | (p[1] & g[0]) | (p[1] & p[0] & cin);
    assign c[3] = g[2] | (p[2] & g[1]) 
                       | (p[2] & p[1] & g[0]) 
                       | (p[2] & p[1] & p[0] & cin);

    // 4. 求和
    assign sum = p ^ c;
endmodule
// =======================================================
// 模块2：16位 CLA 顶层
// =======================================================
module cla_16 (
    input  [15:0] A, B,
    input         Cin,
    output [15:0] Sum,
    output        Cout
);

    // 1. 中间信号
    wire [3:0] P, G;   // 4 个 4-bit 组
    wire [4:0] C;      // 组进位（C0~C4）

    assign C[0] = Cin;

    // 2. 实例化 4 个 4-bit CLA
    cla_4 u0 (A[3:0],    B[3:0],    C[0], Sum[3:0],    P[0], G[0]);
    cla_4 u1 (A[7:4],    B[7:4],    C[1], Sum[7:4],    P[1], G[1]);
    cla_4 u2 (A[11:8],   B[11:8],   C[2], Sum[11:8],   P[2], G[2]);
    cla_4 u3 (A[15:12],  B[15:12],  C[3], Sum[15:12],  P[3], G[3]);

    // 3. 组间进位计算
    assign C[1] = G[0] | (P[0] & C[0]);
    assign C[2] = G[1] | (P[1] & C[1]);
    assign C[3] = G[2] | (P[2] & C[2]);
    assign C[4] = G[3] | (P[3] & C[3]);

    assign Cout = C[4];
endmodule
// =======================================================
// 模块3：24位 CLA 顶层
// =======================================================
module cla_24 (
    input  [23:0] A, B,
    input         Cin,
    output [23:0] Sum,
    output        Cout
);

    // 1. 中间信号
    wire [5:0] P, G;   // 6 个 4-bit 组
    wire [6:0] C;      // 组进位（C0~C6）

    assign C[0] = Cin;

    // 2. 实例化 6 个 4-bit CLA
    cla_4 u0 (A[3:0],    B[3:0],    C[0], Sum[3:0],    P[0], G[0]);
    cla_4 u1 (A[7:4],    B[7:4],    C[1], Sum[7:4],    P[1], G[1]);
    cla_4 u2 (A[11:8],   B[11:8],   C[2], Sum[11:8],   P[2], G[2]);
    cla_4 u3 (A[15:12],  B[15:12],  C[3], Sum[15:12],  P[3], G[3]);
    cla_4 u4 (A[19:16],  B[19:16],  C[4], Sum[19:16],  P[4], G[4]);
    cla_4 u5 (A[23:20],  B[23:20],  C[5], Sum[23:20],  P[5], G[5]);

    // 3. 组间进位（统一格式：G + P·C）
    assign C[1] = G[0] | (P[0] & C[0]);
    assign C[2] = G[1] | (P[1] & C[1]);
    assign C[3] = G[2] | (P[2] & C[2]);
    assign C[4] = G[3] | (P[3] & C[3]);
    assign C[5] = G[4] | (P[4] & C[4]);
    assign C[6] = G[5] | (P[5] & C[5]);

    assign Cout = C[6];
endmodule
```

## 第十二章 多处理器系统 

6. MSI 是最基础的一种 **监听式（snooping）一致性协议**。

| 状态             | 含义                                                         |
| ---------------- | ------------------------------------------------------------ |
| **M (Modified)** | 该 Cache 行被修改过，**只存在于本 CPU Cache 中**，主存是旧的 |
| **S (Shared)**   | 数据是干净的，可被多个 CPU Cache 共享                        |
| **I (Invalid)**  | Cache 中该数据无效                                           |

​	MSI 协议通过 **写独占、读共享、写使他人失效、修改需写回** 的规则，保证多 CPU Cache 中的数据始终一致。**写独占时，其他都要变为I；读共享时，如果其他存在独占，则要退化为共享**。
