题目：[U630592 前缀和非负的排序数 - 洛谷](https://www.luogu.com.cn/problem/U630592)

这是一道**经典的计数型 DP / 组合数学题**，本质和 **括号序列 / Catalan 数** 强相关，但写法略有变化。

## 一、题目在说什么

- 有 **n 个 +1** 和 **n 个 −1**
- 排成一个长度为 `2n` 的序列
- 要求：**任意前缀和 ≥ 0**
- 问：**有多少种不同的排列方式**

### 什么是“前缀和 ≥ 0”

设序列是 `a1, a2, …, a2n`，其中 `ai ∈ {+1, -1}`
定义前缀和：
$$
S_k = a1 + a2 + ... + ak
$$


要求对所有 `k`：$S_k ≥ 0$

## 二、关键等价转化

把：

- `+1` 看成 **左括号 '('**
- `-1` 看成 **右括号 ')'**

那么问题变成：

> 有 n 个 '(' 和 n 个 ')'
> 问有多少个 **合法括号序列**

这正是 **第 n 个 Catalan 数** 👇
$$
C_n = \frac{1}{n+1}\binom{2n}{n}
$$

## 三、但为什么要用 DP？

原因有三点：

1. 题目数据范围：`n ≤ 20`（很小）
2. 直接用公式要处理大组合数、除法，容易出错
3. DP 更直观

## 四、DP 的核心定义

### DP 状态定义

```text
f[i][j]
```

表示：

> **长度为 i 的前缀中，放了 j 个 +1，且前缀和始终 ≥ 0 的方案数**

### 前缀和怎么算？

- 放了 `j` 个 `+1`
- 放了 `i - j` 个 `-1`

前缀和为：
$$
S = j - (i - j) = 2j - i
$$
**合法条件：**

```
2j - i ≥ 0
```

也就是：

```text
j ≥ ceil(i / 2)
```

## 五、DP 转移方程

### 情况 1：第 i 位放 −1

```text
f[i][j] += f[i-1][j]
```

（+1 的数量不变）

### 情况 2：第 i 位放 +1

前提是：前一状态合法：

```text
2 * (j - 1) ≥ i - 1
```

也就是代码中的：

```python
if 2 * (j - 1) >= i - 1:
    f[i][j] += f[i - 1][j - 1]
```

### 合并起来就是：

```text
f[i][j] =
    f[i-1][j]                      // 放 -1
  + f[i-1][j-1]  (如果合法)         // 放 +1
```

## 六、为什么循环范围是这样？

```python
for i in range(1, 2 * n + 1):
    for j in range((i + 1) // 2, i + 1):
```

### 原因：

- `j` 至少要满足：`2j ≥ i`（前缀和非负）
- `(i + 1) // 2` 正好是 `ceil(i/2)`

## 七、初始化与答案位置

### 初始状态

```python
f[0][0] = 1
```

- 空序列，合法，方案数 1

### 最终答案

```python
f[2n][n]
```

- 总长度 `2n`
- 用了 `n` 个 `+1`
- 自动也用了 `n` 个 `-1`

## 八、完整代码

### 答案

```python
n = int(input())

f = [[0] * (2 * n + 2) for _ in range(2 * n + 2)]
f[0][0] = 1

for i in range(1, 2 * n + 1):
    for j in range((i + 1) // 2, i + 1):
        f[i][j] = f[i - 1][j]
        if 2 * (j - 1) >= i - 1:
            f[i][j] += f[i - 1][j - 1]

print(f[2 * n][n])
```

### 伪代码

```
INPUT n

// f[i][j] represents the number of valid sequences of length i
// that contain exactly j occurrences of +1,
// and whose prefix sums are always non-negative
INITIALIZE array f of size (2n + 1) × (2n + 1) with all values set to 0

// Base case: empty sequence is valid
f[0][0] ← 1

FOR i FROM 1 TO 2n DO
    // To keep prefix sum non-negative:
    // 2*j - i ≥ 0  ⇒  j ≥ ceil(i / 2)
    FOR j FROM ceil(i / 2) TO i DO

        // Case 1: place -1 at position i
        // Number of +1 does not change
        f[i][j] ← f[i − 1][j]

        // Case 2: place +1 at position i
        // Previous prefix must still be valid:
        // 2*(j − 1) − (i − 1) ≥ 0
        IF 2 × (j − 1) ≥ i − 1 THEN
            f[i][j] ← f[i][j] + f[i − 1][j − 1]
        END IF

    END FOR
END FOR

// Final result:
// total length is 2n and exactly n occurrences of +1 are used
OUTPUT f[2n][n]
```

👉 **本质是在算 Catalan 数**
