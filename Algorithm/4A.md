### 1. 题目意图分析（通俗版）

**核心任务**：
你是餐厅经理，每天需要 $r_i$ 块新餐巾。
弄来新餐巾有两条路：
1.  **直接买**：贵，但立刻能用。
2.  **洗旧的**：便宜，但要等几天（慢洗/快洗）。

**关键约束**：
*   **物质守恒**：你今天产生多少脏餐巾，就只有多少能拿去洗。
*   **时间流逝**：今天的脏餐巾，洗完变成的是“未来”的新餐巾。

**目标**：
在保证每天大家都有餐巾用的前提下，花最少的钱。

---

### 2. 解题思路：直观的“拆点”模型

为了理清“脏”和“净”的关系，我们将**每一天**拆成两个点：
1.  **脏点 (Node $X_i$)**：代表第 $i$ 天晚上产生的**脏餐巾资源**。
2.  **净点 (Node $Y_i$)**：代表第 $i$ 天早上需要的**干净餐巾需求**。

#### 逐步构建网络图：

**第一步：确立起点与终点 (S 和 T)**
*   **源点 $S$ (供应源)**：代表脏餐巾的产生。
*   **汇点 $T$ (需求坑)**：代表干净餐巾被消耗。

**第二步：描述“脏餐巾产生” ($S \to X_i$)**
*   第 $i$ 天用完了 $r_i$ 块餐巾，变成了脏的。
*   **连边**：$S \to X_i$，容量 $r_i$，费用 $0$。
*   *含义*：系统里每天产生了 $r_i$ 份脏资源供我们支配。

**第三步：描述“干净餐巾需求” ($Y_i \to T$)**
*   第 $i$ 天必须有 $r_i$ 块干净餐巾给客人用。
*   **连边**：$Y_i \to T$，容量 $r_i$，费用 $0$。
*   *含义*：如果流满，说明第 $i$ 天的需求被满足了。

**第四步：描述“如何获得干净餐巾” (连接 $X, Y$ 和 $S$)**

这里有三种方式获得干净餐巾流向 $Y_i$：

1.  **购买新餐巾**：
    *   不需要旧餐巾，直接从商店买。
    *   **连边**：$S \to Y_i$，容量 $\infty$，费用 $p$。
    *   *含义*：只要肯花钱 $p$，源点可以直接供应干净的给 $Y_i$。

2.  **洗涤旧餐巾** (核心：时间旅行)：
    *   **快洗**：第 $i$ 天的脏餐巾 ($X_i$)，洗 $m$ 天，变成第 $i+m$ 天的干净餐巾 ($Y_{i+m}$)。
        *   **连边**：$X_i \to Y_{i+m}$，容量 $\infty$，费用 $f$。
    *   **慢洗**：第 $i$ 天的脏餐巾 ($X_i$)，洗 $n$ 天，变成第 $i+n$ 天的干净餐巾 ($Y_{i+n}$)。
        *   **连边**：$X_i \to Y_{i+n}$，容量 $\infty$，费用 $s$。
    *   *注意*：如果 $i+m > N$，说明洗完活动都结束了，不用连。

3.  **囤积脏餐巾 (延期处理)**：
    *   今天的脏餐巾不洗，留到明天跟明天的脏餐巾一起处理。
    *   **连边**：$X_i \to X_{i+1}$，容量 $\infty$，费用 $0$。
    *   *含义*：脏资源从今天传递到了明天。

**总结**：
跑一次**最小费用最大流**。如果最大流等于总需求 ($\sum r_i$)，说明方案可行，此时的费用就是最小值。

### 3. 复杂度分析

**时间复杂度**：
该算法基于 SPFA 的 EK 增广。
*   **公式**：$O(k \cdot F \cdot E)$
    *   $F$ 是最大流总量（所有 $r_i$ 之和）。
    *   $E$ 是边数（约 $5N$）。
    *   $k$ 是 SPFA 的平均松弛次数（通常很小，约等于 2）。
*   **证明**：在本题中，每次增广至少满足 1 单位的需求。虽然理论上最坏情况 $F$ 很大，但由于本图是分层 DAG 结构（没有负环，且流向明确向后），SPFA 效率极高。对于 $N=2000$ 的数据，实际运行远快于上限，完全可以通过。

**空间复杂度**：
*   我们需要存图（邻接表）。
*   节点数 $V \approx 2N$。
*   边数 $E \approx 5N$（每天有买、存、洗、源、汇等约5条边）。
*   **结果**：$O(N)$，空间占用非常小。

### 什么是“费用最小路径”？它与“增广路”的关系？

在普通的**最大流问题**（Max Flow）中，我们的目标仅仅是“把管道塞满”。
*   **增广路 (Augmenting Path)**：只要是从源点 $S$ 到汇点 $T$ 的、且每一段都还有剩余容量（Capacity > 0）的路径，都叫增广路。
*   在普通最大流算法（如 Edmonds-Karp）中，我们通常用 BFS 找路径。为什么？因为 BFS 找的是“跳数最少”的路径，这样能更快地把流推满，防止死循环，**但它不关心边的“价格”**。

在**最小费用最大流问题**（MCMF）中，情况变了：
*   **不仅要有容量**：路径上的边必须还有剩余容量（同上）。
*   **还要最便宜**：在所有可选的增广路中，我们**必须**优先选择所有边的**费用（Cost）之和最小**的那一条。

**结论**：
这里的 `shortest_path`（SPFA 或 Dijkstra）找到的**就是一条增广路**。
只不过，它不是“距离最短”的增广路，而是**“总单价最低”的增广路**。

> **代码对应**：
> 在 `spfa` 或 `dijkstra` 中，我们累加的是 `dist[u] + e.cost`。这里的 `dist` 实际上代表的是 **“从 S 到当前点每单位流量的累积费用”**。

### 为什么要“不断寻找”？（贪心策略的核心）

这个算法的核心思想叫 **Successive Shortest Path (连续最短路算法)**。这是一个**贪心**策略。

**逻辑推演**：
1.  我想传送流量。
2.  为了总费用最低，我肯定**先挑最便宜的路走**。
3.  我找到了当前最便宜的路（比如单价 5元），但我受限于这条路上的瓶颈容量（比如只有 10 个流量）。
4.  我把这 10 个流量推过去。
5.  现在，那条最便宜的路被塞满了（断了）。
6.  我不得不去寻找**次便宜**的路（比如单价 8元）。
7.  重复这个过程，直到我把所有能走的路都走完，或者达到了最大流的需求。

**这就是 `while (shortest_path(s, t))` 循环的含义**：
每一轮循环，都是在当前的残留网络中，挖掘出一条**当前性价比最高**的路径来利用。

### 代码核心逻辑逐行深究

让我们结合你贴出的代码，详细解释这个过程中的三个关键步骤，特别是**反向边**的玄机。

#### 步骤一：计算推流瓶颈 (Push)

```cpp
// 1. 回溯路径，计算这条路最大能流多少 (push)
long long push = INF;
int curr = t;
while (curr != s) {
    int p = parent_node[curr];  // 父节点
    int idx = parent_edge[curr]; // 边的下标
    // 在路径上所有的边中，找到容量最小的那个，就是瓶颈
    push = min(push, adj[p][idx].cap); 
    curr = p;
}
```
*   **解释**：虽然我们找到了一条“单价”最低的路，但这条路能运多少货，取决于路最窄的地方。比如路上有三段，剩余容量分别是 100, 5, 50，那我们这次最多只能运 5 个单位。

#### 步骤二：累加成本 (Cost Update)

```cpp
// 2. 更新答案
flow += push;
cost += push * dist[t]; 
```
*   **解释**：
    *   `push`：这次运了多少货（比如 5 个）。
    *   `dist[t]`：SPFA/Dijkstra 算出来的，从 S 到 T 走这条最优路径，**每单位**货物的总运费。
    *   总费用 = 流量 $\times$ 单价。

#### 步骤三：更新残留网络 (反向边的奥义)

这是最难理解，也是算法**允许“反悔”**的关键。

```cpp
// 3. 更新残留网络
curr = t;
while (curr != s) {
    int p = parent_node[curr];
    int idx = parent_edge[curr];
    
    // 3.1 正向边容量减少
    adj[p][idx].cap -= push; 
    
    // 3.2 反向边容量增加 (这是关键！)
    int rev_idx = adj[p][idx].rev;
    adj[curr][rev_idx].cap += push;
    
    curr = p;
}
```

**核心问题：为什么要有反向边？为什么反向边的费用是负的？**

假设我们有这么个图：
*   $S \to A$ (费用 0)
*   $A \to T$ (费用 10)
*   $S \to B$ (费用 10)
*   $B \to T$ (费用 0)
*   还有一条路 $A \to B$ (费用 2)

如果算法第一轮贪心，走了 $S \to A \to B \to T$，总费 $0+2+0=2$。挺好。
但如果第一轮因为某种原因走了 $S \to A \to T$ (费用 10)，这显然不是最优的。

**反悔机制**：
当我们在这个正向边 $u \to v$ 上推流后，我们立即增加 $v \to u$ 的容量。
*   正向边 $u \to v$ 的费用是 `cost`。
*   反向边 $v \to u$ 的费用在建图时被设为 `-cost`。

如果在后面的某一轮循环中，SPFA 发现走反向边 $v \to u$ 更划算（比如这会让总路费变小），这意味着什么？
这意味着算法决定：**“退货”**。
*   走 $v \to u$ 意味着把之前从 $u$ 推到 $v$ 的流退回去。
*   费用是负的，意味着**总费用减少了**（把之前花的冤枉钱拿回来了）。

**总结这个逻辑闭环**：
1.  **贪心选择**：每次选当前最便宜的路。
2.  **瓶颈限制**：受限于容量，最便宜的路流满后，不得不选贵的。
3.  **反向修正**：通过负权反向边，算法拥有了“撤销之前决定”的能力。如果发现之前走某条路走错了（导致后面无路可走或太贵），算法可以通过走反向边来调整流量分布，从而保证最终结果一定是**全局最优**。

这就是为什么 `while(shortest_path)` 配合 `update_residual_graph` 能够求出最小费用最大流的全部数学原理。


### 4.通用模板
```cpp
const long long INF = 1e18;
struct Edge {
    int to;
    int rev;        // 反向边在 adj[to] 中的下标
    long long cap;  // 容量
    long long cost; // 费用
};
vector<Edge> adj[MAXN];
long long dist[MAXN];    // 存储从 S 到各点的最小费用
int parent_node[MAXN];   // 记录路径：前驱节点
int parent_edge[MAXN];   // 记录路径：前驱边在 adj 中的下标

// 加边函数
void addEdge(int u, int v, long long cap, long long cost) {
    // 正向边：容量 cap，费用 cost
    adj[u].push_back({v, (int)adj[v].size(), cap, cost});
    // 反向边：容量 0，费用 -cost (关键！这是为了支持“反悔”)
    adj[v].push_back({u, (int)adj[u].size() - 1, 0, -cost});
}

// 求解函数
void minCostMaxFlow(int s, int t, int &flow, long long &cost) {
    // 循环条件：只要还能找到最短路，即最大流问题中的增广路径（spfa 或 dijkstra 返回 true），就继续推流
    while (shortest_path(s, t)) { 
        // 1. 回溯路径，计算这条路最大能流多少 (push)
        long long push = INF;
        int curr = t;
        while (curr != s) {
            int p = parent_node[curr];
            int idx = parent_edge[curr];
            push = min(push, adj[p][idx].cap);
            curr = p;
        }

        // 2. 更新答案
        flow += push;
        cost += push * dist[t]; // 本次增广的费用 = 流量 * 单位路费

        // 3. 更新残留网络 (扣除正向容量，增加反向容量)
        curr = t;
        while (curr != s) {
            int p = parent_node[curr];
            int idx = parent_edge[curr];
            adj[p][idx].cap -= push;
            int rev_idx = adj[p][idx].rev;
            adj[curr][rev_idx].cap += push;
            curr = p;
        }
    }
}
```

SPFA算法核心模板，即求最小费用路径的算法

```cpp
// 这里的最短路径即最小费用的增广路径
bool in_queue[MAXN]; // 防止同一个点重复入队

bool shortest_path(int s, int t) {
    // 1. 初始化距离为无穷大
    for(int i = 0; i < MAXN; i++) dist[i] = INF;
    memset(in_queue, 0, sizeof(in_queue));

    // 2. 源点入队
    dist[s] = 0;
    queue<int> q;
    q.push(s);
    in_queue[s] = true;

    // 3. 开始队列循环
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        in_queue[u] = false; // 出队后标记为不在队列中

        // 遍历所有邻接边
        for (int i = 0; i < adj[u].size(); i++) {
            Edge &e = adj[u][i];
            
            // 核心松弛操作 (Relaxation)
            // 如果这条边还有容量 (e.cap > 0) 且 走这条边到达 v 的费用 (dist[u] + e.cost) 比当前记录的 dist[v] 更小
            if (e.cap > 0 && dist[e.to] > dist[u] + e.cost) {
                
                dist[e.to] = dist[u] + e.cost; // 更新最短距离
                parent_node[e.to] = u;         // 记录前驱，用于回溯路径
                parent_edge[e.to] = i;
                
                // SPFA 的特征：如果 v 被更新了且不在队列里，就把它扔进去
                // 这样 v 可以去更新它的邻居
                if (!in_queue[e.to]) {
                    q.push(e.to);
                    in_queue[e.to] = true;
                }
            }
        }
    }

    // 如果汇点 t 的距离仍为 INF，说明没路了，结束算法
    return dist[t] != INF;
}
```

### 符号说明 (Legend)
*   $G=(V, E)$: 图，包含节点集合 $V$ 和边集合 $E$
*   $r(u, v)$: 边 $(u, v)$ 的**剩余容量** (Residual Capacity)
*   $w(u, v)$: 边 $(u, v)$ 的**单位费用** (Unit Cost)
*   $d[v]$: 从源点 $s$ 到节点 $v$ 的当前最短路径费用
*   $\pi[v]$: 路径的前驱节点 (Predecessor)
*   $Q$: 队列 (Queue)

### 算法 1: SPFA 寻找最短增广路
**Algorithm 1:** $\text{SPFA}(G, s, t)$
**Input:** Residual Graph $G$, Source $s$, Sink $t$
**Output:** Boolean (True if a path exists), updates $d[\cdot]$ and $\pi[\cdot]$

1.  **foreach** $v \in V$ **do**
2.  $\quad d[v] \leftarrow \infty$
3.  $\quad inQueue[v] \leftarrow \text{FALSE}$
4.  $d[s] \leftarrow 0$
5.  $Q.\text{enqueue}(s)$
6.  $inQueue[s] \leftarrow \text{TRUE}$
7.  **while** $Q \neq \emptyset$ **do**
8.  $\quad u \leftarrow Q.\text{dequeue}()$
9.  $\quad inQueue[u] \leftarrow \text{FALSE}$
10. $\quad$ **foreach** neighbor $v$ of $u$ in $E$ **do**
11. $\quad\quad$ $\triangleright$ 松弛操作：仅当边有剩余容量且路径更优时
12. $\quad\quad$ **if** $r(u, v) > 0$ **and** $d[v] > d[u] + w(u, v)$ **then**
13. $\quad\quad\quad d[v] \leftarrow d[u] + w(u, v)$
14. $\quad\quad\quad \pi[v] \leftarrow u$ $\triangleright$ 记录前驱节点用于回溯
15. $\quad\quad\quad$ $\triangleright$ SPFA特征：节点若更新且不在队中，则入队
16. $\quad\quad\quad$ **if** $inQueue[v] = \text{FALSE}$ **then**
17. $\quad\quad\quad\quad Q.\text{enqueue}(v)$
18. $\quad\quad\quad\quad inQueue[v] \leftarrow \text{TRUE}$
19. **return** $(d[t] < \infty)$

---

### 算法 2: 最小费用最大流主过程
**Algorithm 2:** $\text{MinCostMaxFlow}(G, s, t)$
**Input:** Graph $G$, Source $s$, Sink $t$
**Output:** MinCost

1.  $TotalCost \leftarrow 0$
2.  $TotalFlow \leftarrow 0$
3.  $\triangleright$ 只要能找到增广路，就持续循环
4.  **while** $\text{SPFA}(G, s, t)$ is **TRUE** **do**
5.  $\quad \Delta \leftarrow \infty$ $\triangleright$ $\Delta$ 代表本次增广的流量瓶颈
6.  $\quad v \leftarrow t$
7.  $\quad$ $\triangleright$ 步骤1：回溯路径计算最大流
8.  $\quad$ **while** $v \neq s$ **do**
9.  $\quad\quad u \leftarrow \pi[v]$
10. $\quad\quad \Delta \leftarrow \min(\Delta, r(u, v))$
11. $\quad\quad v \leftarrow u$
12. $\quad$ $\triangleright$ 步骤2：更新总结果
13. $\quad TotalFlow \leftarrow TotalFlow + \Delta$
14. $\quad TotalCost \leftarrow TotalCost + \Delta \times d[t]$
15. $\quad$ $\triangleright$ 步骤3：更新残留网络 (反悔机制)
16. $\quad v \leftarrow t$
17. $\quad$ **while** $v \neq s$ **do**
18. $\quad\quad u \leftarrow \pi[v]$
19. $\quad\quad r(u, v) \leftarrow r(u, v) - \Delta$ $\triangleright$ 正向边容量减少
20. $\quad\quad r(v, u) \leftarrow r(v, u) + \Delta$ $\triangleright$ 反向边容量增加
21. $\quad\quad v \leftarrow u$
22. **return** $TotalCost$

### 算法 3: 餐巾计划建图逻辑
**Algorithm 3:** $\text{SolveNapkinPlan}$
**Input:** $N, p, m, f, n, s, R[1 \dots N]$

1.  Initialize graph $G=(V, E)$
2.  $S \leftarrow 0, T \leftarrow 2N + 1$
3.  **for** $i \leftarrow 1$ **to** $N$ **do**
4.  $\quad$ Let $Dirty_i \leftarrow i$ $\triangleright$ 第 $i$ 天晚上（产出脏餐巾）
5.  $\quad$ Let $Clean_i \leftarrow i + N$ $\triangleright$ 第 $i$ 天早上（需求净餐巾）
6.  $\quad$ $\triangleright$ 1. 每天产生脏餐巾 $R[i]$
7.  $\quad$ $\text{AddEdge}(S, Dirty_i, \text{cap}=R[i], \text{cost}=0)$
8.  $\quad$ $\triangleright$ 2. 每天必须满足净餐巾需求 $R[i]$
9.  $\quad$ $\text{AddEdge}(Clean_i, T, \text{cap}=R[i], \text{cost}=0)$
10. $\quad$ $\triangleright$ 3. 购买新餐巾 (随时买，容量无限)
11. $\quad$ $\text{AddEdge}(S, Clean_i, \text{cap}=\infty, \text{cost}=p)$
12. $\quad$ $\triangleright$ 4. 囤积脏餐巾 (留到明天)
13. $\quad$ **if** $i < N$ **then**
14. $\quad\quad \text{AddEdge}(Dirty_i, Dirty_{i+1}, \text{cap}=\infty, \text{cost}=0)$
15. $\quad$ $\triangleright$ 5. 快洗 (需 $m$ 天)
16. $\quad$ **if** $i + m \leq N$ **then**
17. $\quad\quad \text{AddEdge}(Dirty_i, Clean_{i+m}, \text{cap}=\infty, \text{cost}=f)$
18. $\quad$ $\triangleright$ 6. 慢洗 (需 $n$ 天)
19. $\quad$ **if** $i + n \leq N$ **then**
20. $\quad\quad \text{AddEdge}(Dirty_i, Clean_{i+n}, \text{cap}=\infty, \text{cost}=s)$
21. **return** $\text{MinCostMaxFlow}(G, S, T)$

---

### 这里的函数说明：
*   **AddEdge(u, v, cap, cost)**:
    在实际代码中，这步操作等同于：
    1.  添加正向边 $(u, v)$: $r(u, v) = \text{cap}, w(u, v) = \text{cost}$
    2.  添加反向边 $(v, u)$: $r(v, u) = 0, w(v, u) = -\text{cost}$
