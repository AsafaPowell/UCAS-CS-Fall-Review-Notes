题目：[U630586 最大路径权重问题 - 洛谷](https://www.luogu.com.cn/problem/U630586)

这道题可以理解为一个 **“先算最优路径，再想办法削弱它一次”** 的问题。

## 一、题目在说什么

- 给定一个 **n × m 的非负矩阵** `A`
- 从 `(1,1)` 走到 `(n,m)`，**每一步只能向右或向下**
- 一条路径的权值 = 路径上所有格子的数值之和
- 所有可能路径的集合记为 `P`

### 关键变化点

你 **可以任选一个格子 `(p,q)`，把它的值改成 0（只改一个）**

对每一种修改方案 `M`，定义：
$$
F(M) = \max_{\pi \in P} \text{weight}(\pi \mid M)
$$
也就是：

> **在修改之后，所有路径中“最重”的那条路径有多重**

### 最终目标

$$
\min_{M} F(M)
$$

👉 **你要选择“最致命”的一个格子，把它清零，从而尽量降低“最坏情况下的最大路径和”**

## 二、问题的本质

这是一个 **min–max 问题**：

- 外层：你选一个格子清零（min）
- 内层：对手选一条最重的路径（max）

等价理解为：

> 我允许你“砍掉”一个格子的权值，但对手仍然会走一条收益最大的路径，你要让这条路径尽量小。

## 三、为什么这是一个 DP 问题？

### 如果矩阵不修改

经典问题：

> 从 `(1,1)` 到 `(n,m)` 的 **最大路径和**

标准 DP：

```text
dp[i][j] = max(dp[i-1][j], dp[i][j-1]) + a[i][j]
```

时间复杂度：`O(nm)`

## 四、代码详解

### 答案

```python
import sys

# md想到了预处理，但是我是以行列预处理，应该按照对角线

def main():
    input = sys.stdin.read().split()
    # input = "2 2 3 3 6 4".split()
    n, m = int(input[0]), int(input[1])
    a = [[0] * (m + 2) for _ in range(n + 2)]
    
    idx = 2
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            a[i][j] = int(input[idx])
            idx += 1

    # f[i][j]: 从(1,1)到(i,j)的最大路径权重
    f = [[-10**18] * (m + 2) for _ in range(n + 2)]
    f[1][1] = a[1][1]
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if i == 1 and j == 1:
                continue
            f[i][j] = max(f[i-1][j], f[i][j-1]) + a[i][j]

    # g[i][j]: 从(i,j)到(n,m)的最大路径权重
    g = [[-10**18] * (m + 2) for _ in range(n + 2)]
    g[n][m] = a[n][m]
    for i in range(n, 0, -1):
        for j in range(m, 0, -1):
            if i == n and j == m:
                continue
            g[i][j] = max(g[i+1][j], g[i][j+1]) + a[i][j]
            
    diag_max1 = [0] * (n + m + 2)
    diag_max2 = [0] * (n + m + 2)
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            diagv = f[i][j] + g[i][j] - a[i][j]
            if diag_max1[i+j] < diagv:
                diag_max2[i+j] = diag_max1[i+j]
                diag_max1[i+j] = diagv
            else:
                diag_max2[i+j] = max(diag_max2[i+j], diagv)

    ans = 10**18

    for p in range(1, n + 1):
        for q in range(1, m + 1):
            # 经过(p,q)并把它变0的最大分数
            pass_zero = f[p][q] + g[p][q] - 2 * a[p][q]
            
            # 不经过(p,q)的最大分数
            max1, max2 = diag_max1[p+q], diag_max2[p+q]
            if max1 == f[p][q] + g[p][q] - a[p][q]:
                no_pass = max2
            else:
                no_pass = max1
            
            cur_max = max(no_pass, pass_zero)
            
            ans = min(ans, cur_max)

    print(ans)

if __name__ == "__main__":
    main()
```

### 伪代码

```
PROCEDURE MAIN
    READ n, m
    INITIALIZE matrix a of size (n + 2) × (m + 2) with 0

    idx ← 1
    FOR i FROM 1 TO n DO
        FOR j FROM 1 TO m DO
            READ a[i][j]
        END FOR
    END FOR


    // f[i][j]: maximum path weight from (1,1) to (i,j)
    INITIALIZE matrix f of size (n + 2) × (m + 2) with −∞
    f[1][1] ← a[1][1]

    FOR i FROM 1 TO n DO
        FOR j FROM 1 TO m DO
            IF i = 1 AND j = 1 THEN
                CONTINUE
            END IF
            f[i][j] ← max(f[i−1][j], f[i][j−1]) + a[i][j]
        END FOR
    END FOR


    // g[i][j]: maximum path weight from (i,j) to (n,m)
    INITIALIZE matrix g of size (n + 2) × (m + 2) with −∞
    g[n][m] ← a[n][m]

    FOR i FROM n DOWN TO 1 DO
        FOR j FROM m DOWN TO 1 DO
            IF i = n AND j = m THEN
                CONTINUE
            END IF
            g[i][j] ← max(g[i+1][j], g[i][j+1]) + a[i][j]
        END FOR
    END FOR


    // For each diagonal d = i + j,
    // diag_max1[d]: maximum path value through any cell on this diagonal
    // diag_max2[d]: second maximum path value
    INITIALIZE arrays diag_max1 and diag_max2 of size (n + m + 2) with 0

    FOR i FROM 1 TO n DO
        FOR j FROM 1 TO m DO
            diag_value ← f[i][j] + g[i][j] − a[i][j]
            d ← i + j

            IF diag_value > diag_max1[d] THEN
                diag_max2[d] ← diag_max1[d]
                diag_max1[d] ← diag_value
            ELSE
                diag_max2[d] ← max(diag_max2[d], diag_value)
            END IF
        END FOR
    END FOR


    ans ← +∞

    // Enumerate the cell to be set to zero
    FOR p FROM 1 TO n DO
        FOR q FROM 1 TO m DO

            // Case 1: path passes through (p,q) and a[p][q] is set to 0
            pass_zero ← f[p][q] + g[p][q] − 2 × a[p][q]

            // Case 2: path does NOT pass through (p,q)
            d ← p + q
            IF diag_max1[d] = f[p][q] + g[p][q] − a[p][q] THEN
                no_pass ← diag_max2[d]
            ELSE
                no_pass ← diag_max1[d]
            END IF

            // The adversary chooses the worse path
            cur_max ← max(pass_zero, no_pass)

            ans ← min(ans, cur_max)
        END FOR
    END FOR

    OUTPUT ans
END PROCEDURE
```

### 第一步：`f[i][j]` —— 从起点走到任意点

```python
# f[i][j]: 从(1,1)到(i,j)的最大路径权重
f = [[-10**18] * (m + 2) for _ in range(n + 2)]
f[1][1] = a[1][1]

for i in range(1, n + 1):
    for j in range(1, m + 1):
        if i == 1 and j == 1:
            continue
        f[i][j] = max(f[i-1][j], f[i][j-1]) + a[i][j]
```

#### 这一步在干什么？

- 标准 **最大路径和 DP**

- 得到：

  > **所有从 `(1,1)` 到 `(i,j)` 的路径中，最重的一条**

这是之后所有分析的“前半段”。

### 第二步：`g[i][j]` —— 从任意点走到终点

```python
# g[i][j]: 从(i,j)到(n,m)的最大路径权重
g = [[-10**18] * (m + 2) for _ in range(n + 2)]
g[n][m] = a[n][m]

for i in range(n, 0, -1):
    for j in range(m, 0, -1):
        if i == n and j == m:
            continue
        g[i][j] = max(g[i+1][j], g[i][j+1]) + a[i][j]
```

#### 这一步在干什么？

- 和 `f` 对称

- 得到：

  > **所有从 `(i,j)` 到 `(n,m)` 的路径中，最重的一条**

这是路径的“后半段”。

### 关键合成量：路径“经过某点”的最大值

```python
diagv = f[i][j] + g[i][j] - a[i][j]
```

#### 它表示什么？

> **所有经过 `(i,j)` 的路径中，最大的路径和**

因为：

- `f[i][j]`：起点 → `(i,j)`
- `g[i][j]`：`(i,j)` → 终点
- `(i,j)` 被加了两次，要减一次

### 为什么要“按对角线”预处理？

#### 核心几何事实

从 `(1,1)` 到 `(n,m)` 的任意路径：

- 每一步 `(i+j)` 增加 1
- 所以路径在 **每一条对角线 `i+j = 常数` 上恰好经过一个点**

👉 换句话说：

> **路径 = 在每条对角线上选一个点，每条对角线上只能经过一个点**

### 对角线预处理在做什么？

```python
diag_max1 = [0] * (n + m + 2)
diag_max2 = [0] * (n + m + 2)
```

你在做的是：

> **对每一条对角线，记录**
>
> - `diag_max1[d]`：最大的“经过该点的路径和”
> - `diag_max2[d]`：第二大的

代码逻辑：

```python
if diag_max1[i+j] < diagv:
    diag_max2[i+j] = diag_max1[i+j]
    diag_max1[i+j] = diagv
else:
    diag_max2[i+j] = max(diag_max2[i+j], diagv)
```

#### 为什么要存「最大 + 次大」？

因为接下来你要区分：

- 路径 **经过 (p,q)**
- 路径 **绕开 (p,q)**

### 枚举清零点 `(p,q)`：核心 min–max 逻辑

#### 情况 1️⃣：路径经过 `(p,q)`

```python
pass_zero = f[p][q] + g[p][q] - 2 * a[p][q]
```

含义：

- 原来 `(p,q)` 被算两次
- 现在它被清零，两次都要减掉

#### 情况 2️⃣：路径不经过 `(p,q)`

路径在对角线 `p+q` 上**必须选别的点**：

```python
max1, max2 = diag_max1[p+q], diag_max2[p+q]

if max1 == f[p][q] + g[p][q] - a[p][q]:
    no_pass = max2
else:
    no_pass = max1
```

也就是说：

- 如果原最优路径用的不是 `(p,q)` → 不受影响
- 如果原最优路径恰好用 `(p,q)` → 只能退而求其次

------

#### 对手的选择（max）

```python
cur_max = max(no_pass, pass_zero)
```

对手一定选更大的那条路径。

#### 你的选择（min）

```python
ans = min(ans, cur_max)
```

你在所有清零方案中选最小的最坏结果。

## 五、这段代码到底“解决了什么问题”

> **它把“清零一个点后重新算最大路径”的问题，转化为：**
>
> - 一次 DP
> - 一次反向 DP
> - 对角线上的最大 / 次大选择问题

从而把复杂度从：
$$
O((nm)^2)
$$
降到了：
$$
O(nm)
$$
