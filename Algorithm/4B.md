此题与4A均为最小费用最大流问题，在此只阐述建模思路

### 1. 核心思想：满流与“摸鱼”

想象有一条从第 1 天流向第 $n+1$ 天的巨大河流，水的总量是无穷大（代码中的 `INF_FLOW`）。
*   **流量的含义**：这里的每一单位流量，不代表一个人，而是代表**“一个岗位名额”**。
*   **总流量**：假设我们每天理论上都有无数个岗位名额（`INF`）。

在这道题里，我们把“雇佣人干活”和“没人干活”看作两种流向：
1.  **走骨干边（没人干活）**：流量直接从今天流到明天，费用为0。
2.  **走雇佣边（有人干活）**：流量从起聘日跳过中间的日子，直接流到结束日，费用为 $c$。

### 2. 代码中的节点含义

```cpp
// 节点编号：源点 0，天数 1~n+1，汇点 n+2
int S = 0, T = n + 2;
```
*   我们将时间轴上的时刻变成了点。
*   点 $i$ 代表第 $i$ 天的**开始时刻**。
*   点 $i+1$ 代表第 $i$ 天的**结束时刻**（也就是第 $i+1$ 天的开始时刻）。
*   所以，$i \to i+1$ 这条边，就代表了**“度过第 $i$ 天”**这一过程。

### 3. 第一类边：需求限制边（骨干边）

这是代码中最难理解，也是最精彩的一行：

```cpp
// 1. 建立骨干边：i -> i+1，容量为 INF - need[i]，费用 0
for (int i = 1; i <= n; i++) {
    add_edge(i, i + 1, INF_FLOW - need[i], 0);
}
```

**含义解析：**
*   **路径**：$i \to i+1$。
*   **费用**：0（免费）。
*   **容量**：`INF - need[i]`。

**为什么要这么减？**
就像前面说的，总水流是 `INF`。
*   这条边代表**“空缺的岗位”**（也就是这一天**没有**雇人的名额）。
*   如果没有需求，我们可以让所有水流（`INF`）都从这里免费流过去，意味着我们可以一个都不雇。
*   现在题目要求第 $i$ 天至少有 `need[i]` 个人。
*   **建模逻辑**：我们把这条免费的“空缺通道”挤窄！只允许 `INF - need[i]` 的流量通过。
*   **后果**：由于总水流必须保持 `INF` 那么多，而被挤窄的通道过不去，**剩下的 `need[i]` 流量就被“逼”得必须去走别的路**。

**别的路在哪里？就是雇佣边！**

### 4. 第二类边：雇佣方案边（旁路）

```cpp
// 2. 建立雇佣边：s -> t+1，容量 INF，费用 c
for (int i = 0; i < m; i++) {
    // ...
    add_edge(s, t + 1, INF_FLOW, c);
}
```

**含义解析：**
*   **路径**：从 $s$ 直接连到 $t+1$。这相当于架了一座“高架桥”，跨越了从 $s$ 到 $t$ 之间的所有骨干边。
*   **费用**：$c$（雇佣一个人的价格）。
*   **容量**：`INF`（你可以雇无数个人）。

**流量如何运作？**
当第 $k$ 天（$s \le k \le t$）的骨干边被挤窄了（限制了流量），那些原本想走免费通道却过不去的流量，就被迫流向了这座“高架桥”。
*   一旦流量走了 $s \to t+1$ 这条边，它就避开了中间所有 $i \to i+1$ 的限制。
*   这意味着：**这部分流量填补了 $s$ 到 $t$ 之间所有日子的需求缺口**。
*   走这条路是要花钱的，每流过 1 单位流量，就代表雇了一个人，花费 $c$。

### 5. 整体逻辑串联

我们通过一个不等式来证明这个模型的正确性。

对于任意一天 $i$（边 $i \to i+1$）：
$$
\text{流经骨干边的流量} + \text{流经所有覆盖第 } i \text{ 天的雇佣边的流量} = \text{总流量 (INF)}
$$

我们将骨干边的流量代入（因为它被限制了上限）：
$$
(\le \text{INF} - \text{need}[i]) + \text{实际雇佣人数} = \text{INF}
$$

移项得到：
$$
\text{实际雇佣人数} \ge \text{INF} - (\text{INF} - \text{need}[i])
$$
$$
\text{实际雇佣人数} \ge \text{need}[i]
$$

**这就是最小费用最大流在做什么：**
1.  **最大流**：算法试图把 `INF` 这么大的流量从源点推到汇点。为了能流过去，它必须填满所有的需求缺口（因为骨干边不够宽）。
2.  **最小费用**：在被迫要去走“雇佣边”填坑的时候，算法会自动选择最便宜的组合（可能是雇长工，可能是雇短工，取决于哪个 $c$ 更划算）。

### 总结

你的代码并不是模拟“人”的流动，而是模拟 **“岗位空缺”** 的流动。

1.  **总流 INF**：代表总的时间流逝或岗位总池。
2.  **i -> i+1 (容量 INF-a)**：代表**“闲置名额”**。限制这条路，就是强制要求这一天不能全是闲人，必须有 $a$ 个人去干活。
3.  **s -> t+1 (费用 c)**：代表**“干活名额”**。流量从这里走，就意味着这些名额变成了正式员工，从而绕过了那些“限制闲置名额”的关卡，但也付出了费用的代价。

这个模型通过**限制“不干活”的流量，反向逼出了“干活”的最小代价方案**。这就是这道题建模最精妙的地方。
