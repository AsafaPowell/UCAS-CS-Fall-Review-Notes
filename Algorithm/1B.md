题目：[U625832 能耗最小问题 - 洛谷](https://www.luogu.com.cn/problem/U625832)

## 一、题目本质

> 给定一个长度为 $2^n$ 的区间和若干目标点，你可以选择**整段处理**或**对半分治**，问用哪种方式组合，才能让总能耗最小。

这是一个**典型的区间分治 + 最优决策问题**。

## 二、题目理解

### 区间

- 总区间是 $[1, 2^n]$
- 有 `k` 个目标点，位置在区间中

### 对任意区间 `[L, R]`，你有两种操作：

#### ✅ 操作 1：直接处理整个区间

- 若区间内 **没有目标点**
  → 花费 `A`

- 若区间内 **有 cnt 个目标点**
  → 花费
  $$
  B \times cnt \times (R - L + 1)
  $$

#### ✅ 操作 2：继续分割（仅当区间长度 ≥ 2）

- 把区间分成：
  - `[L, mid]`
  - `[mid+1, R]`
- 总能耗 = 左区间能耗 + 右区间能耗

### 目标

> **合理地选择“直接处理”还是“继续分割”，使总能耗最小**

## 三、核心思路

这是一个**标准递归 / 分治 DP**：

我们定义一个函数：

```text
solve(L, R) = 处理区间 [L, R] 的最小能耗
```

### 在区间 `[L, R]` 上的三种情况

#### 1️⃣ 区间内没有目标点

```text
solve(L, R) = A
```

👉 再怎么分都只会更贵，直接处理最好。

#### 2️⃣ 区间内有目标点，且 L == R（区间长度为 1）

```text
solve(L, R) = B × 1 × 1
```

👉 不能再分了，只能直接处理。

#### 3️⃣ 区间内有目标点，且 L < R

你有两种选择：

- **直接处理整个区间**
  $$
  cost_1 = B \times cnt \times (R-L+1)
  $$
  

- **继续对半分**
  $$
  cost_2 = solve(L, mid) + solve(mid+1, R)
  $$
  

取最小值：
$$
solve(L, R) = min(cost_1, cost_2)
$$

## 四、这道题为什么“必须分治”？

- 区间长度最大是 $2^{30}$，不可能枚举区间
- 但目标点只有 $k ≤ 10^5$
- **只有包含目标点的区间才有意义**

👉 用“区间 + 点的分布”递归，**树的规模 ≈ $O(k log 2^n)$**

## 五、代码逐段讲解

### 答案

```python
import sys
import bisect

sys.setrecursionlimit(1 << 25)
input_data = sys.stdin.read().strip().split()
it = iter(input_data)

n = int(next(it))
k = int(next(it))
A = int(next(it))
B = int(next(it))
pts = [int(next(it)) for _ in range(k)]
pts.sort()

def solve(L, R, lo, hi):
    """处理区间[L,R]，pts[lo:hi]是其中的点。"""
    cnt = hi - lo
    if cnt == 0:
        return A
    length = R - L + 1
    direct = B * cnt * length
    if L == R:
        return direct
    mid = (L + R) >> 1
    # 只做一次二分，把当前区间内的点按mid切到左右
    split = bisect.bisect_right(pts, mid, lo, hi)
    left = solve(L, mid, lo, split)
    right = solve(mid + 1, R, split, hi)
    return min(direct, left + right)

total_len = 1 << n
# 初始把全体点限定到[1, 2^n]（其实已保证）
ans = solve(1, total_len,
            bisect.bisect_left(pts, 1),
            bisect.bisect_right(pts, total_len))
print(ans)
```

### 伪代码

```
INPUT n, k, A, B
INPUT array pts[1 .. k]
SORT pts in increasing order

FUNCTION SOLVE(L, R, lo, hi):
    // pts[lo .. hi-1] are the target points inside interval [L, R]
    cnt ← hi - lo

    IF cnt == 0 THEN
        RETURN A

    length ← R - L + 1
    direct_cost ← B × cnt × length

    IF L == R THEN
        RETURN direct_cost

    mid ← (L + R) // 2

    // split points into left and right parts
    split ← first index p in [lo, hi) such that pts[p] > mid

    left_cost ← SOLVE(L, mid, lo, split)
    right_cost ← SOLVE(mid + 1, R, split, hi)

    RETURN min(direct_cost, left_cost + right_cost)
END FUNCTION

total_length ← 2^n

lo0 ← first index i such that pts[i] ≥ 1
hi0 ← first index i such that pts[i] > total_length

ANSWER ← SOLVE(1, total_length, lo0, hi0)
OUTPUT ANSWER
```

### 1️⃣ 读入与预处理

```python
pts = [int(next(it)) for _ in range(k)]
pts.sort()
```

- 对目标点排序
- 后续用 `bisect` 快速切左右区间

### 2️⃣ 递归函数主体

```python
def solve(L, R, lo, hi):
    cnt = hi - lo
```

`cnt` = 当前区间内的目标点数量

#### 情况 1：没有目标点

```python
if cnt == 0:
    return A
```

✔️ 直接处理最优

#### 情况 2：计算直接处理的代价

```python
length = R - L + 1
direct = B * cnt * length
```

#### 情况 3：区间长度为 1

```python
if L == R:
    return direct
```

✔️ 无法再分，只能直接处理

#### 情况 4：继续分治

```python
mid = (L + R) >> 1
split = bisect.bisect_right(pts, mid, lo, hi)
```

- `split` 把当前区间内的点，一次性分成：
  - 左区间：`pts[lo : split]`
  - 右区间：`pts[split : hi]`

#### 递归计算左右区间

```python
left = solve(L, mid, lo, split)
right = solve(mid + 1, R, split, hi)
```

#### 取最小值

```python
return min(direct, left + right)
```

这正是题目要求的“最优处理方式”。
