## 一、题意重述

- 有 **N 头牛** 排成一排

- 初始饥饿度为
  $$
  h_1, h_2, \dots, h_N \quad (h_i \ge 0)
  $$

- 操作：
  **选相邻的两头牛 (i, i+1)**，同时喂它们一袋玉米，使
  $$
  h_i \leftarrow h_i - 1,\quad h_{i+1} \leftarrow h_{i+1} - 1
  $$

- 目标：
  **最终所有牛的饥饿度相同（且非负）**

但 FJ **不知道初始 $h_i$**，只知道
$$
0 \le h_i \le H_i
$$

### 要求

> 统计有多少种满足 $0 \le h_i \le H_i$ 的整数数组
> 使得 **存在一种操作序列**，可以把它们全部变成相同的值
> （答案对 $10^9+7$ 取模）

## 二、核心关键观察

### 1️⃣ 操作的本质

一次操作 = 在某个位置 $i$，对 $i$ 和 $i+1$ **同时 −1**

👉 等价于：

- 对数组做若干次「相邻双点减 1」

### 2️⃣ 最终状态一定是某个统一值 $t$

假设最终所有牛变成：
$$
(t, t, \dots, t)
$$
其中
$$
0 \le t \le \min(h_1,\dots,h_N)
$$

### 3️⃣ 把问题转成「消耗量」

定义：
$$
x_i = h_i - t
$$

- $x_i \ge 0$
- $\sum x_i$ 是总被“吃掉”的量

每一次操作消耗：

- $x_i$ 和 $x_{i+1}$ 各 −1

👉 **每个 $x_i$** 必须完全由与左右的配对操作“消耗完”

### 4️⃣ 关键可行性条件

> 对一维相邻消耗模型，有一个经典结论：

- 如果 **N 是偶数**
  👉 只要满足每个 $x_i \ge 0$，一定可以消耗完，**与 $t$ 无关**。
- 如果 **N 是奇数**
  👉 必须额外枚举最终的 $t$，逐个判断

```cpp
if (N % 2 == 0) {
    cout << f[N][0] << endl;
    return 0;
}
```

如果 **N 是偶数** 能直接提前退出。

## 三、DP 建模思路

### 1️⃣ 枚举最终统一值 $t$

```cpp
for (int t = 0; t <= minH; t++)
```

- 最终所有牛都会被降到 **t**
- 那么原始 $h_i$ 必须 ≥ t

### 2️⃣ 转换为限制条件

$$
x_i = h_i - t \quad \Rightarrow \quad 0 \le x_i \le H_i - t
$$

### 3️⃣ DP 定义

```cpp
f[i][j]
```

含义：

> 考虑前 **i** 头牛
> 第 i 头牛的剩余消耗量为 **j**
> 且前面是 **可配对消耗完的方案数**

### 4️⃣ 状态转移逻辑

相邻牛一起消耗 ⇒ 第 (i-1) 和第 (i) 的消耗量之和有限制：
$$
x_{i-1} + x_i \le H_i - t
$$
所以：
$$
f[i][x_i] = sum_{x_{i-1} = 0}^{H[i]-t-x_i} f[i-1][x_{i-1}]
$$
你代码里用 **前缀和优化**：
$$
prefix[j + 1] = prefix[j] + f[i - 1][j]
$$

$$
f[i][x_i] = prefix[max_{xprev} + 1]
$$



👉 把 $O(NH^2)$ 降成 $O(NH)$

### 5️⃣ 最终合法条件

最后一头牛的剩余消耗量必须是：

```cpp
f[N][0]
```

否则说明还有剩余，无法通过相邻配对消耗完。

## 四、奇偶性为什么这么关键？

### 🔹 N 为偶数

- 相邻配对可以覆盖所有位置
- 本质类似一个“完全匹配”的线性图
- 不需要关心最终 $t$，所有合法 $h$ 都可行

👉 所以 **直接算一次即可**

### 🔹 N 为奇数

- 中间一定会留下一个“无法配对”的位置
- 必须通过调整 t，让这个剩余为 0

👉 所以必须：

```cpp
ans += f[N][0]  // 对所有 t 累加
```

## 五、代码详解

### 答案

```python
MOD = 10**9 + 7

N = int(input())
H = [0] + list(map(int, input().split()))

minH = min(H[1:])
ans = 0

for t in range(minH + 1):
    # f[i][j]: 前 i 头牛，第 i 头牛剩余消耗量为 j 的方案数
    f = [[0] * 1001 for _ in range(N + 1)]
    f[0][0] = 1

    for i in range(1, N + 1):
        prefix = [0] * 1002
        for j in range(1001):
            prefix[j + 1] = (prefix[j] + f[i - 1][j]) % MOD

        for xi in range(H[i] - t + 1):
            max_prev = H[i] - t - xi
            if max_prev < 0:
                break
            f[i][xi] = prefix[max_prev + 1]

    # 偶数 N：直接返回
    if N % 2 == 0:
        print(f[N][0] % MOD)
        exit(0)

    ans = (ans + f[N][0]) % MOD

print(ans)
```

### 伪代码

```
Input N
Input array H[1..N]

minH ← minimum value of H[1..N]
answer ← 0

for t from 0 to minH do
    initialize DP table f[0..N][0..1000] with all zeros
    f[0][0] ← 1

    for i from 1 to N do
        initialize prefix[0..1001]
        prefix[0] ← 0

        for j from 0 to 1000 do
            prefix[j + 1] ← (prefix[j] + f[i - 1][j]) mod MOD
        end for

        for xi from 0 to H[i] - t do
            max_prev ← H[i] - t - xi
            if max_prev < 0 then
                break
            end if
            f[i][xi] ← prefix[max_prev + 1]
        end for
    end for

    if N is even then
        output f[N][0]
        return
    end if

    answer ← (answer + f[N][0]) mod MOD
end for

output answer
```
