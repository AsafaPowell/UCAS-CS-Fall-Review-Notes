题目：[U625836 区间最大值不小于和 - 洛谷](https://www.luogu.com.cn/problem/U625836)

好，这道题**本身不难理解**，真正的难点在于：
👉 **如何把一个看似要枚举所有区间的条件，转化成 O(n) 可检查的条件。**

## 一、题目在说什么

给你一个数组 `a[1..n]`。

要求你判断：

> **对任意子区间 `[i, j]`，该区间里的最大值 ≥ 该区间所有元素之和**

也就是：
$$
\max(a_i, a_{i+1}, \dots, a_j)\ge\sum_{k=i}^{j} a_k
$$
如果**所有**区间都满足，输出 `"YES"`，否则 `"NO"`。

### 一个直觉判断

- 如果一个区间里有 **很多正数**，那“和”可能会很大
- 但最大值只有一个
  👉 **如果区间里“和 > 最大值”，就不合法**

所以本质是在找：

> **是否存在一个区间，使得：区间和 > 区间最大值**

## 二、核心等价转化

我们把“枚举所有区间”这件事换一个角度想。

### 关键观察 ⭐

假设某个区间 `[i, j]` 不合法，那么：

- 设这个区间的最大值是 `a[k]`（i ≤ k ≤ j）

- 则：
  $$
  \sum_{t=i}^{j} a_t > a_k
  $$

👉 **这个区间一定“以某个 a[k] 作为最大值”**

### 所以我们可以换成这样的问题：

> **对每个位置 `k`，检查：是否存在一个包含 `k` 的区间 `[i, j]`，**
>
> - `a[k]` 是该区间的最大值
> - 区间和 `> a[k]`

只要对 **某一个 k** 找到这样的区间 → 直接 `"NO"`。

## 三、固定最大值 k 后，问题变成什么？

固定一个位置 `k`，且假设：

- 在 `[i, j]` 这个区间内
- **所有元素 ≤ a[k]**

👉 那么 `[i, j]` 的左右边界一定被“比 a[k] 大的元素”挡住。

### 所以我们定义：

- `L(k)`：`k` 左边第一个 **严格大于 a[k]** 的位置
- `R(k)`：`k` 右边第一个 **大于等于 a[k]** 的位置

那么所有合法区间 `[i, j]` 必须满足：

```
L(k) < i ≤ k ≤ j < R(k)
```

## 四、如何快速算“区间最大和”

区间和可以用 **前缀和**：
$$
sum(i,j) = P[j+1] - P[i]
$$
为了让 `sum(i,j)` 尽可能大：

- `P[j+1]` 要 **尽量大**
- `P[i]` 要 **尽量小**

### 对每个 k，我们只需要算：

$$
\max_{j \in [k, R-1]} P[j+1]
-
\min_{i \in [L+1, k]} P[i]
$$

如果这个值 **> a[k]** → 存在违规区间。

## 五、算法整体思路$O(n)$

### Step 1：前缀和

```python
P[i] = a[0] + ... + a[i-1]
```

### Step 2：用单调栈算左边最小前缀和

维护一个 **单调递减栈（按 a 值）**：

- 找到每个 `k` 能向左扩展到哪里
- 同时维护该范围内的 `min(P[i])`

得到：

```python
min_P_left[k] = min(P[L+1 ... k])
```

### Step 3：用单调栈算右边最大前缀和

反向扫描：

- 找到右边第一个 `>= a[k]` 的位置
- 同时维护 `max(P[j+1])`

得到：

```python
max_P_right[k] = max(P[k+1 ... R])
```

### Step 4：最终检查

对每个 `k`：

```python
if max_P_right[k] - min_P_left[k] > a[k]:
    print("NO")
```

否则 `"YES"`。

## 六、你的代码在做什么

### 答案

```python
import sys

def solve():
    """
    使用 O(n) 算法解决单个测试用例
    """
    try:
        n_str = sys.stdin.readline()
        if not n_str: return
        n = int(n_str)
        a = list(map(int, sys.stdin.readline().split()))
    except (IOError, ValueError):
        return

    # 1. 计算前缀和数组 P
    # P[i] 存储 a[0]...a[i-1] 的和
    P = [0] * (n + 1)
    for i in range(n):
        P[i + 1] = P[i] + a[i]

    # 2. 计算每个 k 左侧区间的 prefix_sum 最小值
    # min_P_left[k] = min(P[i]) for i in [prev_greater[k] + 1, k]
    min_P_left = [0] * n
    stack_left = []  # 存储 (index, min_P_in_its_range)
    for k in range(n):
        # 我们要找的 sum(a[i..j]) = P[j+1] - P[i]
        # P[i] 的 i 范围是 [L+1, k]，所以我们计算 min(P[L+1]...P[k])
        # 注意 P[k] 是 a[0..k-1] 的和，所以我们取 P[k]
        current_min_p = P[k]
        while stack_left and a[stack_left[-1][0]] <= a[k]:
            # 栈顶元素比当前元素小，它的管辖范围被当前元素k吞并
            # 我们需要继承它范围内的最小值
            _, popped_min_p = stack_left.pop()
            current_min_p = min(current_min_p, popped_min_p)
        min_P_left[k] = current_min_p
        stack_left.append((k, current_min_p))

    # 3. 计算每个 k 右侧区间的 prefix_sum 最大值
    # max_P_right[k] = max(P[j+1]) for j in [k, next_greater_equal[k] - 1]
    max_P_right = [0] * n
    stack_right = [] # 存储 (index, max_P_in_its_range)
    for k in range(n - 1, -1, -1):
        # 我们要找的 P[j+1] 的 j+1 范围是 [k+1, R]
        # 所以我们计算 max(P[k+1]...P[R])
        current_max_p = P[k + 1]
        while stack_right and a[stack_right[-1][0]] < a[k]:
            # 注意这里是 <, 因为我们要找 next_greater_equal, 对称地，栈里维护 < a[k] 的
            _, popped_max_p = stack_right.pop()
            current_max_p = max(current_max_p, popped_max_p)
        max_P_right[k] = current_max_p
        stack_right.append((k, current_max_p))
        
    # 4. 最终检查
    for k in range(n):
        # 检查是否存在一个子区间的和大于其最大值 a[k]
        # max_sum = max_P_right[k] - min_P_left[k]
        if max_P_right[k] - min_P_left[k] > a[k]:
            print("NO")
            return
            
    # 如果循环结束都没有找到反例
    print("YES")


def main():
    """
    主函数，处理多个测试用例
    """
    try:
        t_str = sys.stdin.readline()
        if not t_str: return
        t = int(t_str)
        for _ in range(t):
            solve()
    except (IOError, ValueError):
        return

if __name__ == "__main__":
    main()
```

### 伪代码

```
procedure MAIN
    read t
    repeat t times
        read n
        read array a[0 .. n-1]

        // compute prefix sums
        P[0] ← 0
        for i ← 0 to n-1 do
            P[i+1] ← P[i] + a[i]

        // compute min prefix sum on the left for each k
        initialize empty stack S_left   // each element: (index, min_prefix)
        for k ← 0 to n-1 do
            current_min ← P[k]
            while S_left is not empty AND a[S_left.top.index] ≤ a[k] do
                popped ← S_left.pop()
                current_min ← min(current_min, popped.min_prefix)
            min_P_left[k] ← current_min
            push (k, current_min) into S_left

        // compute max prefix sum on the right for each k
        initialize empty stack S_right  // each element: (index, max_prefix)
        for k ← n-1 down to 0 do
            current_max ← P[k+1]
            while S_right is not empty AND a[S_right.top.index] < a[k] do
                popped ← S_right.pop()
                current_max ← max(current_max, popped.max_prefix)
            max_P_right[k] ← current_max
            push (k, current_max) into S_right

        // check validity for all k
        valid ← true
        for k ← 0 to n-1 do
            if max_P_right[k] - min_P_left[k] > a[k] then
                valid ← false
                break

        if valid then
            output "YES"
        else
            output "NO"
```

### 1️⃣ 前缀和

```python
P[i + 1] = P[i] + a[i]
```

标准做法，用于 $O(n)$ 求区间和。

### 2️⃣ 左边最小前缀和（单调栈）

```python
while stack_left and a[stack_left[-1][0]] <= a[k]:
    ...
```

含义：

- **比当前 `a[k]` 小或相等的元素不可能挡住 k**
- 它们的区间范围会被 `k` 吞并
- 所以要继承它们的 `min P`

### 3️⃣ 右边最大前缀和（单调栈）

```python
while stack_right and a[stack_right[-1][0]] < a[k]:
```

这里是 `<` 而不是 `<=`：

- 为了实现 **“右边第一个 ≥ a[k]”**
- 保证左右边界对称、不重不漏

### 4️⃣ 核心判定

```python
if max_P_right[k] - min_P_left[k] > a[k]:
```

这正是：
$$
\max \text{区间和} > \text{区间最大值}
$$
一旦成立 → 不可能满足题目要求。
