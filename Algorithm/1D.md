题目：[U625894 “中间数”计数问题 - 洛谷](https://www.luogu.com.cn/problem/U625894)

这是一道**看似“模拟集合构造”，实则是“规律 + 递归/分治”**的题。

## 一、题目在做什么

- 初始：集合 $S={1}$
- 给定正整数 $N$
- 重复以下操作，直到做不了为止：

在区间 $[1,N]$ 中找所有正整数 $x$​，使得
$$
\min_{y\in S} |x-y| \ \text{尽可能大且} \ >1
$$

- 若这样的 $x$ 唯一，加入 $S$
- 若有多个，选**最小的**加入
- 当再也找不到满足“最小距离 > 1”的 $x$ 时，停止

**输出最终集合 $S$ 的大小**

## 二、核心理解：这其实是在“最大化最小距离”

当前集合 $S$ 把区间 $[1,N]$ 切成若干段空区间。
对每个候选点 $x$，它到 $S$ 的最近距离就是：

- 落在某个空区间里，距离 = 到最近端点的距离

**每一步**：

- 选择“离已有点最远”的位置（最大化最小距离）
- 距离必须 **严格大于 1**
- 距离相同选最小的 $x$

这就是典型的 **“区间二分/中点优先”** 过程。

## 三、关键等价：并不需要真的模拟加点

直接模拟会很慢（$N$ 可很大）。
但注意到：

- 初始只有 1
- 每次都会在“当前最长的空区间”里选一个**中点附近**的点
- 这个过程与**递归地把区间一分为二**是等价的

👉 **最终集合大小，只与区间长度有关，与具体位置无关**

## 四、边界情况（直接来自规则）

```python
if r <= 1: return 1
if r == 2: return 1
if r == 3: return 2
```

解释：

- (r=1)：只有 1 → `{1}` → 1 个
- (r=2)：只能 `{1}`（2 距离 1 为 1，不 >1）→ 1 个
- (r=3)：可以加入 3 → `{1,3}` → 2 个

## 五、递归公式

令：

```python
mid = (r + 1) // 2
```

这相当于选“当前最优的中点”。

### 1️⃣ 当 $r$ 为奇数

区间被**对称地**分成两段同长度的子区间：

```
长度 r
→ 左：mid-1
→ 右：mid-1
```

但中点本身被加入一次，所以：
$$
dfs(r) = 2 \cdot dfs(mid) - 1
$$

### 2️⃣ 当 $r$ 为偶数

左右子区间长度不完全相同：

```
→ 左：mid-1
→ 右：mid
```

对应公式：
$$
dfs(r) = dfs(mid) + dfs(mid+1) - 1
$$
减 1 的原因：

- 左右递归都“算”了中点的贡献
- 实际中点只加入一次

## 六、代码逐行解释

### 答案

```python
import sys
sys.setrecursionlimit(1000000)
from functools import lru_cache

@lru_cache(maxsize=None)
def dfs(r: int) -> int:
    # 处理递归的边界情况
    if r <= 1:
        return 1
    if r == 2:
        return 1
    if r == 3:
        return 2
    # 根据奇偶性使用不同的递归公式
    mid = (r + 1) // 2
    if r % 2 == 0:  # 偶数情况
        return dfs(mid) + dfs(mid + 1) - 1
    else:          # 奇数情况
        return 2 * dfs(mid) - 1

# 主函数读取输入并输出结果
def main():
    n = int(sys.stdin.readline().strip())
    print(dfs(n))

if __name__ == "__main__":
    main()

```

## 七、为什么这个递推是对的

- 每次加入的点，都是把当前区间“切开”
- 后续行为在左右区间中**相互独立**
- 但中点只算一次，所以要减 1
- 整个过程就是一棵**近似完全二叉递归树**
