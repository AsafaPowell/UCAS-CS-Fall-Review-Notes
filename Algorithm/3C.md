## 1) 题意分析（把规则翻译成“你能做什么/不能做什么”）

你有 N 件物品，分两类：

* **A 类（普通）**：只能直接卖，卖出得到现金 `P`。
* **B 类（可升级）**：有两种卖法

  * 不升级直接卖：得到 `P1`
  * 先花固定升级费 `C` 升级（升级本身不产生收入），再卖：得到 `P2`（且题目保证 `P2 > P1`）

过程约束：

* **初始现金 = 0**
* 必须把所有物品都卖掉
* 操作顺序任意，但要满足：

  * 卖东西是收入（现金增加）
  * 升级要先付 `C`（现金必须 ≥ C 才能升级）

目标：通过选择卖法 + 安排顺序，使**最终现金最大**。

这题的“坑”在于：哪怕升级后更赚，你也可能**一开始没钱付 C**，需要先卖一些东西凑启动资金。

---

## 2) 为什么这样解决（从直觉到可证明的建模）

### 2.1 “升级值不值”先算清楚：Δ（净增益）

对每个 B 类物品：

* 不升级：最终贡献 `P1`
* 升级：先付 `C` 再卖得 `P2`，净贡献 `P2 - C`

升级相对不升级多赚多少？

$[
\Delta = (P2 - C) - P1 = P2 - P1 - C
]$

* 若 **Δ ≤ 0**：升级不划算（甚至亏），最优一定“不升级当普通卖”
* 若 **Δ > 0**：升级理论上更赚，是“优质升级物品”

代码里把 Δ>0 的 B 物品收集进 `goodItems`，Δ≤0 的直接当“永远不升级”。

---

### 2.2 一个关键性质：只要能付得起“第一次升级费”，后面就不会再卡住

对 **优质升级物品**（Δ>0）：

$[
\Delta > 0 \Rightarrow P2 > P1 + C \Rightarrow P2 > C
]$

这句特别重要：**升级后卖出一次，你手里会有至少 P2 的现金，而 P2 > C，所以你立刻就能付得起下一次升级费。**

因此：

* 真正的难点只在于：**能不能凑出第一次升级所需的 C 现金（启动资金）**
* 一旦能启动，后续所有“你决定要升级的优质物品”都能一路升级下去，不会中途断档。

这就是为什么题目最终会变成“先凑启动资金”的选择问题。

---

### 2.3 先定义一个“基准方案”：所有东西都不升级卖掉

把所有物品都用“无需预付款”的方式卖掉（A卖P，B卖P1），总收入：

$[
baseTotal = \sum A(P) + \sum B(P1)
]$

代码里的 `baseTotal` 就是这个。

它有两个含义：

1. **如果你完全不升级，答案就是 baseTotal**
2. **第一次升级之前，你最多也只能通过“不升级卖”来获得现金**
   所以如果 `baseTotal < C`：就算把所有东西都不升级卖完，也凑不出 C，**一次升级都做不了** → 直接输出 `baseTotal`。

代码的情况 2 正是这个判断。

---

### 2.4 如果升级都做得到，最理想是“把所有优质升级物品都升级”

假设你能顺利启动升级，那么“优质升级物品”每个都能多赚 Δ，于是理想总收益：

$[
baseTotal + \sum(\Delta_{good})
]$

代码里 `totalProfit` 是所有优质 Δ 的和。

---

### 2.5 启动资金从哪里来：S0（不牺牲利润也能拿到的钱）

有些钱你**无论最优怎么做**都可以先拿到，不会影响最终最优值：

* 所有 A 类物品（反正只能直接卖）
* 所有 Δ≤0 的 B 类物品（反正最优不升级）

把这些“必然直接卖”的钱加起来：

$[
S0 = \sum A(P) + \sum B_{\Delta \le 0}(P1)
]$

代码里 `S0` 就是这个“无损启动资金”。

于是：

* 若 `S0 >= C`：你可以先把这些必卖物品卖掉，立刻凑够 C 启动升级，然后把所有优质物品全部升级 → 直接输出 `baseTotal + totalProfit`（代码情况 3）

---

### 2.6 最困难的情况：S0 < C ≤ baseTotal（能升级，但需要“牺牲”一些优质物品来凑首笔 C）

当 `S0 < C` 时，光靠“必卖物品”不够付第一次升级费。

你就必须从“优质升级物品”里挑一些**不升级先卖掉**，用它们的 `P1` 来补缺口：

$[
D = C - S0 \quad (D>0)
]$

但这么做会损失它们本来能带来的额外利润 Δ。

所以问题变成：

> 从优质物品里选一批“牺牲升级”的物品，
> 使它们的 `P1` 之和 ≥ D（凑够启动资金缺口），
> 同时让损失的 Δ 之和最小。

这就是一个典型的 **0/1 背包**：

* 每个优质物品：

  * “重量/贡献”：`P1`（提供的启动资金）
  * “代价”：`Δ`（放弃升级造成的损失）
* 目标：达到至少 D 的总 `P1`，使总代价最小

最终答案：

$[
ans = baseTotal + totalProfit - \text{minLost}
]$

其中 `minLost` 是为凑够启动资金而不得不放弃的最小 Δ 总和。

> 为什么不用贪心？因为这是“达到阈值的子集选择”，很容易出现“看起来最划算的先选”却导致需要多选一个大损失物品的情况，背包才稳。

## 3) 代码如何实现上述思路（逐段对应）

### 3.1 输入解析（区分一行 1 个数 / 2 个数）

题目每行可能是：

* `P`（A 类）
* `P1 P2`（B 类）

所以代码用 `getline + stringstream` 把整行拆成整数数组 `v`，根据 `v.size()` 判断类型。

### 3.2 统计 baseTotal、S0，并挑出 goodItems

* A 类：

  * `baseTotal += P`
  * `S0 += P`
* B 类：

  * `baseTotal += P1`
  * `delta = P2 - P1 - C`
  * 若 `delta <= 0`：`S0 += P1`（当作必卖）
  * 否则：放入 `goodItems.push_back({P1, delta})`，并累加 `totalProfit += delta`

### 3.3 三个“可直接输出”的简单分支

1. `goodItems.empty()`：没有优质升级，升级没意义 → `baseTotal`
2. `baseTotal < C`：连一次升级都启动不了 → `baseTotal`
3. `S0 >= C`：无损启动，全部优质升级 → `baseTotal + totalProfit`

### 3.4 进入背包：S0 < C ≤ baseTotal

缺口：

```cpp
int D = upgradeCost - (int)S0;
```

背包 dp 定义（非常关键）：

* `dp[s]`：选择一些优质物品“不升级”，使它们的 `P1` 之和（**截断到 ≤ C**）为 `s` 时，**损失 Δ 的最小值**
* 初始化：`dp[0] = 0`，其余 INF

为什么“截断到 C”？

* 你只关心“是否凑够 D”，而 D ≤ C
* 任何 `P1` 总和超过 C 的状态，对“能否启动”没有额外价值，可以统一当成 C，减少状态数

转移（0/1 背包倒序）：

```cpp
for each item (P1, delta):
  for s from C down to 0:
    t = min(C, s + P1)
    dp[t] = min(dp[t], dp[s] + delta)
```

最后在 `s ∈ [D, C]` 里取最小损失：

```cpp
minLost = min(dp[s]) for s>=D
ans = baseTotal + totalProfit - minLost
```

## 4) 复杂度（含证明思路）

设：

* `G = goodItems.size()`，显然 `G ≤ N ≤ 1000`
* `C ≤ 5000`

### 时间复杂度

背包双循环：

* 外层遍历每个优质物品：`G`
* 内层遍历 `s = 0..C`：`C+1`

所以：

$[
T = O(G \cdot C) \le O(N \cdot C) \le 1000 \cdot 5000 = 5\times 10^6
]$

再加上读入和常数判断，整体仍是 `O(NC)`。

**证明要点**：dp 的每个状态更新是 O(1)，总更新次数是 `G*(C+1)`。

### 空间复杂度

只用一维数组 `dp[0..C]`：

$[
S = O(C)
]$

其余 `goodItems` 存 G 个 pair：`O(G)`，但主导仍是 `O(C + G)`，在约束内非常小。

## 5) 保留核心逻辑的伪代码（对应你的实现）

```text
read N, C
baseTotal = 0
S0 = 0
goodItems = empty list
totalProfit = 0

for i in 1..N:
    read one line as numbers v
    if v has 1 number:      # type A
        P = v[0]
        baseTotal += P
        S0 += P
    else:                   # type B
        P1 = v[0], P2 = v[1]
        baseTotal += P1
        delta = P2 - P1 - C
        if delta <= 0:
            S0 += P1            # never upgrade
        else:
            append (P1, delta) to goodItems
            totalProfit += delta

if goodItems is empty:
    output baseTotal
    stop

if baseTotal < C:
    output baseTotal
    stop

if S0 >= C:
    output baseTotal + totalProfit
    stop

# hard case: need sacrifice some good items to start
D = C - S0

dp[0..C] = INF
dp[0] = 0

for each (P1, delta) in goodItems:
    for s from C down to 0:
        if dp[s] == INF: continue
        t = s + P1
        if t > C: t = C
        dp[t] = min(dp[t], dp[s] + delta)

minLost = min(dp[s]) for s in [D..C]
ans = baseTotal + totalProfit - minLost
output ans
```

### 第二段：最大化利润 (Knapsack-like Optimization)

**Algorithm 2:** Maximize Profit with Initial Constraints  
**Input:** Number of items $N$, cost threshold $C$, item data  
**Output:** Maximum profit or base total

1.  $baseTotal \leftarrow 0$, $S_0 \leftarrow 0$
2.  $goodItems \leftarrow \emptyset$, $totalProfit \leftarrow 0$
3.  $\triangleright$ *Process items: Type A adds to base; Type B adds potential upgrades*
4.  **for** $i \leftarrow 1$ **to** $N$ **do**
5.  $\quad$ Read item $v$
6.  $\quad$ **if** $v$ has 1 number ($P$) **then**
7.  $\quad\quad$ $baseTotal \leftarrow baseTotal + P$
8.  $\quad\quad$ $S_0 \leftarrow S_0 + P$
9.  $\quad$ **else** ($P_1, P_2$)
10. $\quad\quad$ $baseTotal \leftarrow baseTotal + P_1$
11. $\quad\quad$ $\Delta \leftarrow P_2 - P_1 - C$
12. $\quad\quad$ **if** $\Delta \le 0$ **then**
13. $\quad\quad\quad$ $S_0 \leftarrow S_0 + P_1$
14. $\quad\quad$ **else**
15. $\quad\quad\quad$ Append $(P_1, \Delta)$ to $goodItems$
16. $\quad\quad\quad$ $totalProfit \leftarrow totalProfit + \Delta$
17. $\quad\quad$ **end if**
18. $\quad$ **end if**
19. **end for**
20. $\triangleright$ *Handle edge cases and immediate validity checks*
21. **if** $goodItems = \emptyset$ **or** $baseTotal < C$ **then**
22. $\quad$ **return** $baseTotal$
23. **if** $S_0 \ge C$ **then**
24. $\quad$ **return** $baseTotal + totalProfit$
25. $\triangleright$ *Hard case: Solve constrained optimization using DP (Sacrifice min profit to bridge gap)*
26. $D \leftarrow C - S_0$
27. Initialize $dp[0 \dots C] \leftarrow \infty$, $dp[0] \leftarrow 0$
28. **for each** $(P_1, \Delta)$ **in** $goodItems$ **do**
29. $\quad$ **for** $s \leftarrow C$ **down to** $0$ **do**
30. $\quad\quad$ **if** $dp[s] = \infty$ **then** **continue**
31. $\quad\quad$ $t \leftarrow \min(s + P_1, C)$
32. $\quad\quad$ $dp[t] \leftarrow \min(dp[t], dp[s] + \Delta)$
33. $\quad$ **end for**
34. **end for**
35. $minLost \leftarrow \min(\{dp[s] \mid D \le s \le C\})$
36. **return** $baseTotal + totalProfit - minLost$

```cpp
#include <bits/stdc++.h>
using namespace std;
int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int N, upgradeCost; // N: 物品数, upgradeCost: 每次升级费用 C
    if (!(cin >> N >> upgradeCost))
        return 0;
    // 为了区分“这一行有 1 个数还是 2 个数”，用 getline + stringstream 解析
    string line;
    getline(cin, line);               // 吃掉第一行剩余的换行
    long long baseTotal = 0;          // 所有物品都不升级时的总收入 base
    long long S0 = 0;                 // 类型 A + 所有 Δ<=0 的 B 的总 P/P1（初始可用资金）
    vector<pair<int, int>> goodItems; // 记录所有“优质升级”物品 (P1, Δ>0)
    long long totalProfit = 0;        // 所有优质升级物品的 Δ 之和
    for (int i = 0; i < N; ++i)
    {
        // 读一整行，然后拆成若干整数
        do
        {
            if (!getline(cin, line))
                break;
        } while (line.size() == 0); // 防止空行
        stringstream ss(line);
        vector<int> v;
        int x;
        while (ss >> x)
            v.push_back(x);
        if (v.size() == 1)
        {
            // 类型 A：只能直接卖，价值 P
            int P = v[0];
            baseTotal += P;
            S0 += P; // 一开始就能直接卖掉
        }
        else if (v.size() == 2)
        {
            // 类型 B：可升级 (P1, P2)
            int P1 = v[0], P2 = v[1];
            baseTotal += P1;                   // “完全不升级”方案下，该物品贡献 P1
            int delta = P2 - P1 - upgradeCost; // 相对“不升级”的净增益 Δ
            if (delta <= 0)
            {
                // 劣质升级：无论如何都不值得升级，当成类型 A 使用
                S0 += P1;
            }
            else
            {
                // 优质升级：理论上应该升级，先记录下来
                goodItems.push_back({P1, delta});
                totalProfit += delta;
            }
        }
        else
        {
            // 输入保证不会这样，这里只是防御性处理
            continue;
        }
    }
    // 情况 1：根本没有优质升级物品 → 不可能有收益更好的方案
    if (goodItems.empty())
    {
        cout << baseTotal << "\n";
        return 0;
    }
    // 情况 2：即便所有物品都不升级卖掉，钱也不够一次升级费用 → 无法升级
    if (baseTotal < upgradeCost)
    {
        cout << baseTotal << "\n";
        return 0;
    }
    // 情况 3：只靠“永不升级”的那部分钱就能买到第一份升级费用 → 所有优质物品都升级
    if (S0 >= upgradeCost)
    {
        cout << baseTotal + totalProfit << "\n";
        return 0;
    }
    // 情况 4：困难档：S0 < C <= baseTotal
    // 需要从优质物品中挑一些“不升级”，用它们的 P1 补足缺口
    int D = upgradeCost - (int)S0; // 启动资金缺口 D > 0，且 D <= upgradeCost <=
    5000 const long long INF = (1LL << 60);
    // dp[s]：选一部分优质物品“不升级”，使得它们 P1 之和（截断到 <=C）为 s 时，
    // 最小的损失 Δ 之和
    vector<long long> dp(upgradeCost + 1, INF);
    dp[0] = 0;
    for (auto &item : goodItems)
    {
        int P1 = item.first;
        int delta = item.second;
        // 0/1 背包：倒序遍历 s
        for (int s = upgradeCost; s >= 0; --s)
        {
            if (dp[s] == INF)
                continue;
            int t = s + P1;
            if (t > upgradeCost)
                t = upgradeCost; // 超过 C 无意义，统一截断到 C
            long long nd = dp[s] + delta;
            if (nd < dp[t])
                dp[t] = nd;
        }
    }
    long long minLost = INF;
    for (int s = D; s <= upgradeCost; ++s)
    {
        if (dp[s] < minLost)
            minLost = dp[s];
    }
    // 理论上在 baseTotal >= C 时一定能凑到，但保险起见做个兜底
    if (minLost == INF)
    {
        cout << baseTotal << "\n";
        return 0;
    }
    long long ans = baseTotal + totalProfit - minLost;
    cout << ans << "\n";
    return 0;
}
```

## 6) 补充：帮助复习掌握的关键点/常见坑

### 6.1 这题为什么“只卡第一次升级”？

因为你只把 **Δ>0** 的物品称为“优质升级”，它们满足 `P2 > C`，升级卖出后现金一定 > C，能连续升级下去。
如果你把 Δ≤0 的也当“可升级候选”，这个性质就不成立了，所以代码先做了筛选。

### 6.2 背包 dp 的“意义”要背牢

* 你不是在最大化价值，而是在**最小化损失**
* 约束不是“重量 ≤ 容量”，而是“重量 ≥ 缺口 D”
* 用“截断到 C”的技巧，把“≥C”的所有状态合并成一个，省状态、好写实现

### 6.3 一个能让贪心翻车的直观例子（说明为什么要 dp）

设 `C=10, S0=0`，优质物品：

* 物品1：`P1=10, Δ=9`
* 物品2：`P1=6,  Δ=4`
* 物品3：`P1=6,  Δ=4`

缺口 D=10。
如果贪心“先选 P1 最大”会选物品1，损失 9；
但 dp 会发现选 2+3，P1=12（截断也行），损失 8，更优。

### 6.4 long long 的必要性

* `N` 最多 1000，单价最多 10000，总和可到 1e7 级别其实 int 也够
  但 dp 里 INF、以及更稳妥的累计（尤其你用 `1LL<<60`）都用 long long 是正确习惯。
