## 一、题目重新建模

### 1️⃣ 题目在做什么？

- 有 `n` 本书，**顺序不能变**
- 每本书厚度（长度）是 `h[i]`
- 书架可以分成若干 **层**
- 每一层：
  - 放的是 **连续的一段书**
  - 这一层的**宽度 = 这一层中书的最大厚度**
  - 这一层中书的**厚度和 ≤ m**
- 总书架宽度 = **所有层的宽度之和**
- 目标：**最小化总宽度**

## 二、DP 的核心思路

### 2️⃣ DP 状态定义

```cpp
dp[i] = 放完前 i 本书，书架的最小总宽度
```

### 3️⃣ 状态转移公式

假设最后一层放的是书 `j+1 ... i`，那么：

- 这一层宽度 = `max(h[j+1...i])`
- 前面的书代价 = `dp[j]`

所以：

```text
dp[i] = min over j<i 且 sum(h[j+1...i]) ≤ m:
        dp[j] + max(h[j+1...i])
```

⚠️ 这就是问题的本质
⚠️ 但这个式子**直接算是 O(n²)**，会超时

## 三、代码如何把 O(n²) 优化到 O(n log n)

关键在于两点：

1. **滑动窗口**：控制 `sum(h[j+1...i]) ≤ m`
2. **单调栈 + 线段树**：高效维护
   `dp[j] + max(h[j+1...i])`

## 四、代码详解

### 答案

```python
import sys

def solve():
    # 优化读取
    input_data = sys.stdin.read().split()
    if not input_data:
        return
    
    n = int(input_data[0])
    m = int(input_data[1])
    h = list(map(int, input_data[2:]))
    h = [0] + h  # 使下标从 1 开始匹配原 C++ 代码

    # 线段树参数：Python 中迭代式线段树通常使用 2 的幂次大小
    size = 1
    while size <= n + 1:
        size *= 2
    
    inf = float('inf')
    # min_dp[v] 存储 dp[j]
      = [inf] * (2 * size)
    # min_val[v] 存储 dp[j] + max_h
    min_val = [inf] * (2 * size)
    # lazy_tag[v] 存储区间更新的 max_h
    lazy = [-1] * (2 * size)

    def push_up(v):
        min_dp[v] = min(min_dp[2 * v], min_dp[2 * v + 1])
        min_val[v] = min(min_val[2 * v], min_val[2 * v + 1])

    def apply(v, h_val):
        min_val[v] = min_dp[v] + h_val
        lazy[v] = h_val

    def push_down(v):
        if lazy[v] != -1:
            apply(2 * v, lazy[v])
            apply(2 * v + 1, lazy[v])
            lazy[v] = -1

    # 为了处理方便，这里使用递归式区间更新（迭代式带懒标记较复杂）
    # 但 Python 递归深度需注意
    sys.setrecursionlimit(200000)

    def update_dp(pos, val):
        # 单点更新 dp 序列
        idx = pos + size
        # 下传从根到叶子的路径（虽是单点，但为了维护一致性）
        for s in range(size.bit_length(), 0, -1):
            push_down(idx >> s)
        
        min_dp[idx] = val
        # 向上回溯更新
        while idx > 1:
            idx >>= 1
            push_up(idx)

    def update_range_h(l, r, new_h):
        if l > r: return
        
        def _update(v, tl, tr, l, r, new_h):
            if l <= tl and tr <= r:
                apply(v, new_h)
                return
            push_down(v)
            tm = (tl + tr) // 2
            if l <= tm:
                _update(2 * v, tl, tm, l, r, new_h)
            if r > tm:
                _update(2 * v + 1, tm + 1, tr, l, r, new_h)
            push_up(v)

        _update(1, 0, size - 1, l, r, new_h)

    def query(l, r):
        if l > r: return inf
        
        def _query(v, tl, tr, l, r):
            if l <= tl and tr <= r:
                return min_val[v]
            push_down(v)
            tm = (tl + tr) // 2
            res = inf
            if l <= tm:
                res = min(res, _query(2 * v, tl, tm, l, r))
            if r > tm:
                res = min(res, _query(2 * v + 1, tm + 1, tr, l, r))
            return res

        return _query(1, 0, size - 1, l, r)

    dp = [0] * (n + 1)
    update_dp(0, 0)
    
    stk = []
    L = 1
    current_sum = 0
    
    for i in range(1, n + 1):
        # 1. 维护滑动窗口
        current_sum += h[i]
        while current_sum > m:
            current_sum -= h[L]
            L += 1
        
        # 2. 维护单调栈并更新区间 max_h
        while stk and h[stk[-1]] <= h[i]:
            stk.pop()
        
        prev_idx = stk[-1] if stk else 0
        # 更新 [prev_idx, i-1] 范围内的 max_h 为 h[i]
        update_range_h(prev_idx, i - 1, h[i])
        stk.append(i)
        
        # 3. 查询 dp[i] = min(dp[j] + max(h[j+1...i])) 其中 L-1 <= j < i
        # 注意原 C++ 代码 L 是从 0 开始自增，Python 逻辑保持一致：j 范围 [L-1, i-1]
        dp[i] = query(L - 1, i - 1)
        
        # 4. 更新线段树中的 dp 值
        update_dp(i, dp[i])

    print(dp[n])

solve()
```

### 伪代码

```cc
ALGORITHM SequencePartition(n, m, h)
    // n: number of elements, m: max sum limit, h: array of heights
    
    // Initialize DP and Segment Tree
    dp[0] ← 0
    FOR i FROM 1 TO n: dp[i] ← INF
    
    // tree structure stores:
    // - tree.min_dp: stores the value of dp[j]
    // - tree.min_val: stores the combined value (dp[j] + max(h[j+1...i]))
    // - tree.lazy: lazy propagation tag for range max_h updates
    Initialize SegmentTree with size 4 * n
    SegmentTree.UpdateDP(0, 0)
    
    stack ← EmptyStack()      // Monotonic stack to maintain decreasing heights
    L ← 1                     // Left boundary for the sliding window sum
    current_sum ← 0
    
    FOR i FROM 1 TO n:
        // 1. Maintain sliding window such that sum(h[L...i]) <= m
        current_sum ← current_sum + h[i]
        WHILE current_sum > m:
            current_sum ← current_sum - h[L]
            L ← L + 1
        
        // 2. Update Segment Tree using Monotonic Stack
        // The current h[i] affects intervals where it is the new maximum height
        WHILE stack is not empty AND h[stack.top()] <= h[i]:
            stack.pop()
        
        // prev_idx is the index of the first element to the left greater than h[i]
        prev_idx ← stack.empty() ? 0 : stack.top()
        
        // For all split points j in [prev_idx, i-1], the max height of 
        // the new partition [j+1...i] is now h[i]
        SegmentTree.UpdateRangeMaxH(prev_idx, i - 1, h[i])
        
        stack.push(i)
        
        // 3. Query the optimal DP value for current i
        // Valid split points j must be in range [L-1, i-1]
        dp[i] ← SegmentTree.QueryMinVal(L - 1, i - 1)
        
        // 4. Store dp[i] into the tree for future partitions
        SegmentTree.UpdateDP(i, dp[i])
        
    RETURN dp[n]
END ALGORITHM
```

### 线段树整体设计（非常重要）

#### 1️⃣ size 的含义

```python
size = 1
while size <= n + 1:
    size *= 2
```

- 线段树底层数组大小
- 能覆盖索引 `[0 .. n]`
- 用的是**完全二叉树结构**

#### 2️⃣ 三个核心数组

```python
min_dp  # 存 dp[j]
min_val # 存 dp[j] + 当前区间最大高度
lazy    # 懒标记：这个区间统一用的 max_h
```

#### 含义对应关系

| 数组         | 物理意义                     |
| ------------ | ---------------------------- |
| `min_dp[j]`  | 以 j 作为分割点的 dp[j]      |
| `min_val[j]` | dp[j] + max(h[j+1..当前 i])  |
| `lazy[v]`    | 这个区间当前统一使用的 max_h |

⚠️ 你查询的 **永远是 `min_val`**

### 线段树的三个基本操作

#### 1️⃣ push_up

```python
def push_up(v):
    min_dp[v] = min(min_dp[2 * v], min_dp[2 * v + 1])
    min_val[v] = min(min_val[2 * v], min_val[2 * v + 1])
```

👉 标准线段树合并
👉 保证父节点保存区间最优解

#### 2️⃣ apply（区间统一 max_h）

```python
def apply(v, h_val):
    min_val[v] = min_dp[v] + h_val
    lazy[v] = h_val
```

#### 这是整道题最核心的一句

它表达的是：

> 对这个区间内所有 j：
> **max(h[j+1..i]) = h_val**

所以：

```text
dp[j] + max = dp[j] + h_val
```

#### 3️⃣ push_down

```python
def push_down(v):
    if lazy[v] != -1:
        apply(2 * v, lazy[v])
        apply(2 * v + 1, lazy[v])
        lazy[v] = -1
```

👉 延迟标记下传
👉 保证子区间的 max_h 正确

## 五、单点更新 dp[j]

```python
def update_dp(pos, val):
    idx = pos + size
```

### 语义

> 设置 dp[pos] = val

### 为什么要 push_down？

```python
for s in range(size.bit_length(), 0, -1):
    push_down(idx >> s)
```

因为：

- 这个位置可能在某个“统一 max_h 区间”里
- 必须先把懒标记下传
- 再安全地改 dp 值

### 更新并回溯

```python
min_dp[idx] = val
while idx > 1:
    idx >>= 1
    push_up(idx)
```

👉 更新 dp
👉 保证整棵树一致

## 六、区间更新 max_h（单调栈配套）

```python
def update_range_h(l, r, new_h):
```

### 语义

> 对所有 j ∈ [l, r]
> **max(h[j+1..i]) = new_h**

### 递归实现

```python
def _update(v, tl, tr, l, r, new_h):
```

- 覆盖整段 → `apply`
- 否则下传 + 分裂
- 更新完后 `push_up`

👉 完整实现了：

```text
min_val[j] = dp[j] + new_h
```

## 七、区间查询（真正算 dp[i] 的地方）

```python
def query(l, r):
```

### 语义

```text
min(dp[j] + max(h[j+1..i]))，j ∈ [l, r]
```

### 为什么返回 `min_val`？

因为：

```text
min_val[j] = dp[j] + max(h[j+1..i])
```

正是 dp 转移所需的值

## 八、主循环：一轮 i 到底发生了什么

```python
dp = [0] * (n + 1)
update_dp(0, 0)
```

👉 初始化：
**dp[0] = 0**，表示“什么书都不放，宽度为 0”

### 1️⃣ 滑动窗口（厚度和约束）

```python
current_sum += h[i]
while current_sum > m:
    current_sum -= h[L]
    L += 1
```

### 保证：

```text
sum(h[L..i]) ≤ m
```

所以：

```text
合法分割点 j ≥ L-1
```

### 2️⃣ 单调栈：维护“谁是最大值”

```python
while stk and h[stk[-1]] <= h[i]:
    stk.pop()
```

### 含义

- 新书更厚
- 之前比它薄的书
- **不可能再成为最大值**
- 全部淘汰

### 得到区间左端点

```python
prev_idx = stk[-1] if stk else 0
```

👉 对所有：

```text
j ∈ [prev_idx, i-1]
```

都有：

```text
max(h[j+1..i]) = h[i]
```

### 3️⃣ 区间更新 max_h

```python
update_range_h(prev_idx, i - 1, h[i])
```

🔥 这一句是整道题的**“最大值维护核心”**

### 4️⃣ 查询 dp[i]

```python
dp[i] = query(L - 1, i - 1)
```

### 完整数学意义

```text
dp[i] = min(
    dp[j] + max(h[j+1..i])
), j ∈ [L-1, i-1]
```

- `L-1`：保证厚度和 ≤ m
- `i-1`：至少放一本

### 5️⃣ 把 dp[i] 写回线段树

```python
update_dp(i, dp[i])
```

👉 供未来的 dp[k] 使用

## 九、为什么这套代码是 $O(n log n)$

|   部分   |      复杂度      |
| :------: | :--------------: |
| 滑动窗口 |      $O(n)$      |
|  单调栈  |      $O(n)$      |
| 区间更新 |    $O(log n)$    |
| 区间查询 |    $O(log n)$    |
|   总体   | **$O(n log n)$** |
